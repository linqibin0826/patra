# Create Orchestrator Test

## Context

You are creating a **test for an application layer orchestrator** in Papertrace.

Orchestrators coordinate domain objects and external dependencies. Tests must mock all dependencies and verify orchestration logic.

### Target Orchestrator

**Class path**: $ARGUMENTS

If not provided, ask user for orchestrator class path (e.g., `com.patra.ingest.app.usecase.plan.PlanIngestionOrchestrator`).

### Testing Strategy

**App layer orchestrators**:
- ✅ Mock all dependencies (repositories, ports, services)
- ✅ Test orchestration flow (steps in correct order)
- ✅ Test transaction boundaries (`@Transactional` methods)
- ✅ Test error handling and exception translation
- ✅ Test domain event publication
- ❌ DO NOT test business logic (that's in domain layer)

### Read Standards

Testing patterns: @docs/DEV-GUIDE.md (section "Testing Strategies")

## Your Task

Create a comprehensive orchestrator test following Papertrace patterns.

### Step 1: Analyze Orchestrator

Read the target orchestrator class to understand:

1. **Dependencies**: What is injected via constructor?
   - Repositories (domain ports)
   - External ports (registry, external services)
   - Other services

2. **Use case methods**: What are the main orchestration methods?
   - Typically: `execute(Command cmd)` → `Result`

3. **Orchestration steps**: What happens in each method?
   - Load domain objects
   - Execute domain behavior
   - Persist changes
   - Publish events
   - Return result

### Step 2: Create Test Class

```java
package com.patra.{service}.app.usecase.{feature};

import org.junit.jupiter.api.Test;
import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Nested;
import org.junit.jupiter.api.extension.ExtendWith;
import org.mockito.InjectMocks;
import org.mockito.Mock;
import org.mockito.junit.jupiter.MockitoExtension;

import static org.assertj.core.api.Assertions.*;
import static org.mockito.Mockito.*;
import static org.mockito.ArgumentMatchers.*;

/**
 * Unit tests for {@link TargetOrchestrator}.
 *
 * Tests orchestration logic with mocked dependencies.
 * Does NOT test business logic (that's in domain tests).
 *
 * @author {generated by Claude Code}
 * @since 0.1.0
 */
@ExtendWith(MockitoExtension.class)
@DisplayName("TargetOrchestrator Tests")
class TargetOrchestratorTest {

    @Mock
    private DomainRepository mockRepository;

    @Mock
    private ExternalPort mockExternalPort;

    @Mock
    private EventPublisher mockEventPublisher;

    @InjectMocks
    private TargetOrchestrator orchestrator;

    private Command testCommand;
    private DomainAggregate testAggregate;

    @BeforeEach
    void setUp() {
        // Prepare test data
        testCommand = new Command("test-code", OperationCode.HARVEST);
        testAggregate = DomainAggregate.create("test-code");

        // Setup default mock behaviors
        when(mockRepository.findById(any())).thenReturn(Optional.of(testAggregate));
        when(mockRepository.save(any())).thenAnswer(inv -> inv.getArgument(0));
        when(mockExternalPort.fetchConfig(any(), any())).thenReturn(createTestConfig());
    }

    @Nested
    @DisplayName("Happy Path Tests")
    class HappyPathTests {

        @Test
        @DisplayName("should execute complete orchestration flow successfully")
        void shouldExecuteCompleteFlow() {
            // Given
            // (setUp provides default test data)

            // When
            Result result = orchestrator.execute(testCommand);

            // Then - Verify result
            assertThat(result).isNotNull();
            assertThat(result.getStatus()).isEqualTo("SUCCESS");

            // Then - Verify orchestration steps (in order)
            InOrder inOrder = inOrder(mockRepository, mockExternalPort, mockEventPublisher);

            // Step 1: Load config
            inOrder.verify(mockExternalPort).fetchConfig(any(), any());

            // Step 2: Load aggregate
            inOrder.verify(mockRepository).findById(any());

            // Step 3: Persist changes
            inOrder.verify(mockRepository).save(any(DomainAggregate.class));

            // Step 4: Publish events
            inOrder.verify(mockEventPublisher).publish(anyList());
        }

        @Test
        @DisplayName("should pass correct parameters to dependencies")
        void shouldPassCorrectParameters() {
            // When
            orchestrator.execute(testCommand);

            // Then - Verify external port called with correct params
            verify(mockExternalPort).fetchConfig(
                eq(testCommand.getProvenanceCode()),
                eq(testCommand.getOperationCode())
            );

            // Then - Verify repository called with correct aggregate
            verify(mockRepository).save(argThat(agg ->
                agg.getCode().equals(testCommand.getCode()) &&
                agg.getStatus() == ExpectedStatus.COMPLETED
            ));
        }

        @Test
        @DisplayName("should publish domain events from aggregate")
        void shouldPublishDomainEvents() {
            // Given
            DomainEvent expectedEvent = new SomeEvent(1L, "test", Instant.now());
            testAggregate.addDomainEvent(expectedEvent);

            // When
            orchestrator.execute(testCommand);

            // Then
            verify(mockEventPublisher).publish(argThat(events ->
                events.size() == 1 &&
                events.get(0).eventType().equals("expected.event.type")
            ));
        }
    }

    @Nested
    @DisplayName("Error Handling Tests")
    class ErrorHandlingTests {

        @Test
        @DisplayName("should throw exception when aggregate not found")
        void shouldThrowWhenAggregateNotFound() {
            // Given
            when(mockRepository.findById(any())).thenReturn(Optional.empty());

            // When & Then
            assertThatThrownBy(() -> orchestrator.execute(testCommand))
                .isInstanceOf(AggregateNotFoundException.class)
                .hasMessageContaining("not found");

            // Then - Verify no save or publish happened
            verify(mockRepository, never()).save(any());
            verify(mockEventPublisher, never()).publish(any());
        }

        @Test
        @DisplayName("should handle external service failure gracefully")
        void shouldHandleExternalServiceFailure() {
            // Given
            when(mockExternalPort.fetchConfig(any(), any()))
                .thenThrow(new ExternalServiceException("Service unavailable"));

            // When & Then
            assertThatThrownBy(() -> orchestrator.execute(testCommand))
                .isInstanceOf(OrchestrationException.class)
                .hasCauseInstanceOf(ExternalServiceException.class);
        }

        @Test
        @DisplayName("should rollback on domain validation failure")
        void shouldRollbackOnValidationFailure() {
            // Given
            Command invalidCommand = new Command(null, OperationCode.HARVEST);

            // When & Then
            assertThatThrownBy(() -> orchestrator.execute(invalidCommand))
                .isInstanceOf(ValidationException.class);

            // Then - Verify no persistence happened (due to @Transactional rollback)
            verify(mockRepository, never()).save(any());
        }
    }

    @Nested
    @DisplayName("Edge Case Tests")
    class EdgeCaseTests {

        @Test
        @DisplayName("should handle empty domain events list")
        void shouldHandleEmptyEvents() {
            // Given
            testAggregate.clearDomainEvents();

            // When
            orchestrator.execute(testCommand);

            // Then - Should not call publisher with empty list
            verify(mockEventPublisher, never()).publish(any());
        }

        @Test
        @DisplayName("should handle multiple domain events")
        void shouldHandleMultipleEvents() {
            // Given
            testAggregate.addDomainEvent(new Event1(...));
            testAggregate.addDomainEvent(new Event2(...));
            testAggregate.addDomainEvent(new Event3(...));

            // When
            orchestrator.execute(testCommand);

            // Then
            verify(mockEventPublisher).publish(argThat(events ->
                events.size() == 3
            ));
        }
    }

    @Nested
    @DisplayName("Transaction Boundary Tests")
    class TransactionTests {

        @Test
        @DisplayName("should pull events AFTER save (within transaction)")
        void shouldPullEventsAfterSave() {
            // When
            orchestrator.execute(testCommand);

            // Then - Verify order: save happens before event publication
            InOrder inOrder = inOrder(mockRepository, mockEventPublisher);
            inOrder.verify(mockRepository).save(any());
            inOrder.verify(mockEventPublisher).publish(any());
        }

        @Test
        @DisplayName("should NOT pull events if save fails")
        void shouldNotPullEventsIfSaveFails() {
            // Given
            when(mockRepository.save(any()))
                .thenThrow(new PersistenceException("DB error"));

            // When & Then
            assertThatThrownBy(() -> orchestrator.execute(testCommand))
                .isInstanceOf(PersistenceException.class);

            // Then - Events should not be published
            verify(mockEventPublisher, never()).publish(any());
        }
    }

    // Helper methods

    private ConfigSnapshot createTestConfig() {
        return new ConfigSnapshot(
            ProvenanceCode.TEST,
            new HttpConfig(Duration.ofSeconds(30), Map.of()),
            new RetryConfig(3, Duration.ofSeconds(1)),
            new RateLimitConfig(100, 200)
        );
    }
}
```

### Step 3: Key Testing Patterns

#### A. Verify Orchestration Order

```java
// Use InOrder to verify steps happen in sequence
InOrder inOrder = inOrder(mock1, mock2, mock3);
inOrder.verify(mock1).method1();
inOrder.verify(mock2).method2();
inOrder.verify(mock3).method3();
```

#### B. Verify Method Arguments

```java
// Verify exact arguments
verify(mock).method(eq(expectedArg));

// Verify argument properties
verify(mock).method(argThat(arg ->
    arg.getProperty() == expectedValue
));

// Capture and assert
ArgumentCaptor<Aggregate> captor = ArgumentCaptor.forClass(Aggregate.class);
verify(mock).save(captor.capture());
assertThat(captor.getValue().getStatus()).isEqualTo(COMPLETED);
```

#### C. Verify Event Publication

```java
// Verify events published
verify(mockEventPublisher).publish(argThat(events ->
    events.size() == 2 &&
    events.get(0) instanceof ExpectedEvent1 &&
    events.get(1) instanceof ExpectedEvent2
));
```

### Step 4: Test Coverage Checklist

Ensure tests cover:

- ✅ Happy path (complete orchestration flow)
- ✅ Orchestration order (steps in correct sequence)
- ✅ Parameter passing (correct args to dependencies)
- ✅ Domain event publication (events published after save)
- ✅ Error handling (aggregate not found, external service failure)
- ✅ Transaction boundaries (rollback on error)
- ✅ Edge cases (empty events, multiple events)

**Target**: 80%+ coverage for orchestrator

### Step 5: Anti-Patterns to Avoid

❌ **DON'T test business logic in orchestrator tests**:
```java
// BAD: Testing business rule (should be in domain test)
@Test
void shouldTransitionToDraftStatus() {
    orchestrator.execute(command);
    assertThat(aggregate.getStatus()).isEqualTo(DRAFT);
}
```

❌ **DON'T use real dependencies**:
```java
// BAD: Using real repository
private DomainRepository repository = new DomainRepositoryImpl();
```

❌ **DON'T test implementation details**:
```java
// BAD: Testing private method behavior
verify(mockRepository, times(3)).findById(any());  // Who cares how many times?
```

✅ **DO test orchestration logic**:
```java
// GOOD: Verify orchestration steps
InOrder inOrder = inOrder(mockRepository, mockPort);
inOrder.verify(mockPort).fetchConfig(...);
inOrder.verify(mockRepository).save(...);
```
