## 🎯 核心概念速查表

### 关键组件

| 组件 | 层次 | 职责 | 泛型支持 |
|------|------|------|----------|
| `DataType` | Domain | 类型标记枚举 | ❌ |
| `TypeReference<T>` | Domain | 保持泛型信息 | ✅ |
| `DataSourcePort` | Domain | 数据获取契约 | ✅ 泛型方法 |
| `DataSourceAdapter` | Infrastructure | 桥接 Domain-Framework | ✅ |
| `ProviderRegistry` | Infrastructure | 二维索引管理 | ❌ |
| `DataSourceProvider` | Framework | 数据源提供者 | ✅ 泛型方法 |
| `DataProcessor<T>` | Framework | 策略化处理器 | ✅ 泛型类 |

### 类型关系

```
DataType (枚举)
    ↓ 包含
Class<?> (数据类型的 Class)
    ↓ 运行时验证
TypeReference<T> (类型引用)
    ↓ 保持泛型
T (实际数据类型)
```

### 索引结构

```
ProviderRegistry:
    主索引: Map<(provenanceCode, dataType), Provider>
    辅助索引 1: Map<provenanceCode, Set<DataType>>
    辅助索引 2: Map<dataType, List<Provider>>

ProcessorRegistry:
    主索引: Map<dataType, Processor>
```

#### 1.3 Framework 层：创建 Processor

```java
// src/main/java/com/patra/starter/provenance/xxx/processor/YourDataProcessor.java
@Slf4j
@Component
public class YourDataProcessor implements DataProcessor<YourDataClass> {

    private final YourDataConverter converter;

    @Override
    public DataType getDataType() {
        return DataType.YOUR_TYPE;
    }

    @Override
    public ProcessResult<YourDataClass> process(
            ProviderRequest request,
            ProviderContext context) {

        try {
            // 1. 获取客户端
            YourClient client = context.getClient(YourClient.class);

            // 2. 调用 API
            YourApiResponse response = client.fetchData(
                request.executionParams().query()
            );

            // 3. 转换数据
            List<YourDataClass> dataList = response.getItems().stream()
                .map(this::transform)
                .filter(data -> validate(data).isValid())
                .collect(Collectors.toList());

            // 4. 返回结果
            return ProcessResult.success(dataList, response.getNextCursor());

        } catch (Exception ex) {
            log.error("数据处理失败", ex);
            return ProcessResult.failure("处理失败: " + ex.getMessage());
        }
    }

    @Override
    public YourDataClass transform(Object rawData) throws TransformationException {
        if (!(rawData instanceof YourApiData)) {
            throw new TransformationException("类型错误");
        }
        return converter.convert((YourApiData) rawData);
    }

    @Override
    public ValidationResult validate(YourDataClass data) {
        List<String> errors = new ArrayList<>();
        if (data.getId() == null) {
            errors.add("ID 不能为空");
        }
        return errors.isEmpty()
            ? ValidationResult.success()
            : ValidationResult.failure(errors);
    }
}
```

#### 2.2 创建 Provider

```java
@Slf4j
@Component
public class NewSourceDataSourceProvider implements DataSourceProvider {

    private static final String PROVENANCE_CODE = "new_source";
    private static final Set<DataType> SUPPORTED_TYPES = Set.of(
        DataType.YOUR_TYPE
    );

    private final NewSourceClient client;
    private final ProvenanceProperties properties;
    private final Map<DataType, DataProcessor<?>> processors;

    public NewSourceDataSourceProvider(
            NewSourceClient client,
            ProvenanceProperties properties,
            ProcessorRegistry processorRegistry) {
        this.client = client;
        this.properties = properties;
        this.processors = SUPPORTED_TYPES.stream()
            .filter(processorRegistry::supports)
            .collect(Collectors.toMap(
                Function.identity(),
                processorRegistry::getProcessor
            ));
    }

    @Override
    public String getProvenanceCode() {
        return PROVENANCE_CODE;
    }

    @Override
    public Set<DataType> getSupportedDataTypes() {
        return SUPPORTED_TYPES;
    }

    @Override
    @SuppressWarnings("unchecked")
    public <T> ProviderResult<T> fetchData(
            ProviderRequest request,
            DataType dataType,
            Class<T> targetClass) {

        if (!supports(dataType)) {
            return ProviderResult.failure(
                dataType,
                "不支持的数据类型",
                ProviderResult.ErrorType.NON_RETRIABLE
            );
        }

        DataProcessor<?> processor = processors.get(dataType);

        ProviderContext context = ProviderContext.builder()
            .config(properties.mergeWithRuntime(PROVENANCE_CODE, request.config()))
            .client(client)
            .batchMetadata(request.metadata())
            .build();

        ProcessResult<?> processResult = processor.process(request, context);

        if (processResult.success()) {
            List<T> typedData = (List<T>) processResult.data();
            return ProviderResult.<T>builder()
                .success(true)
                .data(typedData)
                .dataType(dataType)
                .nextCursorToken(processResult.nextCursor())
                .fetchedCount(typedData.size())
                .errorType(ProviderResult.ErrorType.NONE)
                .build();
        } else {
            return ProviderResult.failure(
                dataType,
                processResult.errorMessage(),
                ProviderResult.ErrorType.RETRIABLE
            );
        }
    }
}
```

### 场景 3: Application 层调用

#### 3.1 基本调用

```java
@Component
@RequiredArgsConstructor
public class DataHarvester {

    private final DataSourcePort dataSourcePort;

    public void harvest(String provenanceCode, DataType dataType) {
        // 构建上下文
        ExecutionContext context = ExecutionContext.builder()
            .provenanceCode(provenanceCode)
            .operationCode("HARVEST")
            .compiledQuery("query string")
            .build();

        // 创建类型引用（根据 dataType 动态创建）
        TypeReference<?> typeRef = createTypeReference(dataType);

        // 批次
        Batch batch = Batch.builder()
            .batchNo(1)
            .pageSize(100)
            .build();

        // 调用
        DataFetchResult<?> result = dataSourcePort.fetchData(
            context,
            dataType,
            typeRef,
            batch
        );

        // 处理结果
        if (result.success()) {
            log.info("获取成功: {} 条数据", result.fetchedCount());
            processData(result.data(), dataType);
        } else {
            log.error("获取失败: {}", result.errorMessage());
        }
    }

    private TypeReference<?> createTypeReference(DataType dataType) {
        Class<?> dataClass = dataType.getDataClass();
        // 使用反射或工厂创建 TypeReference
        // 简化示例（实际需要更复杂的处理）
        return new TypeReference<Object>() {
            @Override
            public Class<Object> getRawType() {
                return (Class<Object>) dataClass;
            }
        };
    }
}
```

#### 3.2 批量采集多种类型

```java
@Component
public class MultiTypeHarvester {

    private final DataSourcePort dataSourcePort;

    public void harvestAllTypes(String provenanceCode) {
        // 查询支持的类型
        Set<DataType> supportedTypes = dataSourcePort.getSupportedTypes(provenanceCode);

        // 并行采集
        supportedTypes.parallelStream().forEach(dataType -> {
            try {
                harvestByType(provenanceCode, dataType);
            } catch (Exception ex) {
                log.error("采集失败: source={}, type={}", provenanceCode, dataType, ex);
            }
        });
    }

    private void harvestByType(String provenanceCode, DataType dataType) {
        // 根据 dataType 创建对应的 TypeReference
        switch (dataType) {
            case LITERATURE -> harvestLiterature(provenanceCode);
            case JOURNAL -> harvestJournal(provenanceCode);
            case DRUG -> harvestDrug(provenanceCode);
            // ...
        }
    }

    private void harvestLiterature(String provenanceCode) {
        TypeReference<CanonicalLiterature> typeRef = new TypeReference<>() {};
        DataFetchResult<CanonicalLiterature> result = dataSourcePort.fetchData(
            buildContext(provenanceCode),
            DataType.LITERATURE,
            typeRef,
            Batch.firstBatch()
        );
        // 处理文献数据
    }
}
```

## 🔍 故障排查检查清单

### 问题 1: Provider 未找到

**错误信息**: `未找到 Provider: source=xxx, type=yyy`

**检查步骤**:
1. ✅ Provider 类是否添加了 `@Component` 注解？
2. ✅ Provider 的 `getSupportedDataTypes()` 是否包含该类型？
3. ✅ Provider 所在包是否被 Spring 扫描？
4. ✅ 启动日志中是否有 "注册 Provider" 的日志？

**解决方案**:
```java
// 确保 Provider 正确注册
@Component  // ← 必须有
public class MyProvider implements DataSourceProvider {

    private static final Set<DataType> SUPPORTED_TYPES = Set.of(
        DataType.YOUR_TYPE  // ← 必须包含
    );

    @Override
    public Set<DataType> getSupportedDataTypes() {
        return SUPPORTED_TYPES;
    }
}
```

### 问题 2: Processor 未找到

**错误信息**: `未找到 Processor: type=xxx`

**检查步骤**:
1. ✅ Processor 类是否添加了 `@Component` 注解？
2. ✅ Processor 的 `getDataType()` 是否返回正确的类型？
3. ✅ ProcessorRegistry 日志中是否注册了该 Processor？

**解决方案**:
```java
@Component  // ← 必须有
public class MyProcessor implements DataProcessor<MyData> {

    @Override
    public DataType getDataType() {
        return DataType.YOUR_TYPE;  // ← 必须匹配
    }
}
```

### 问题 3: 类型不匹配

**错误信息**: `类型不匹配: DataType 期望 X, TypeReference 提供 Y`

**检查步骤**:
1. ✅ DataType 枚举中的 `dataClass` 是否正确？
2. ✅ TypeReference 的泛型参数是否正确？

**解决方案**:
```java
// DataType 定义
YOUR_TYPE("code", "desc", YourDataClass.class, "table");
                          // ↑ 必须匹配

// 调用时
TypeReference<YourDataClass> typeRef = new TypeReference<YourDataClass>() {};
              // ↑ 必须匹配
```

### 问题 4: 数据转换失败

**错误信息**: 部分数据转换失败

**检查步骤**:
1. ✅ `transform()` 方法是否有异常处理？
2. ✅ `validate()` 方法是否正确验证？
3. ✅ 日志中是否有详细的错误信息？

**解决方案**:
```java
@Override
public ProcessResult<T> process(...) {
    List<T> dataList = new ArrayList<>();
    List<String> failedIds = new ArrayList<>();

    for (RawData raw : rawDataList) {
        try {
            T data = transform(raw);
            ValidationResult validation = validate(data);

            if (validation.isValid()) {
                dataList.add(data);
            } else {
                failedIds.add(raw.getId());
                log.warn("验证失败: id={}, errors={}",
                    raw.getId(), validation.getErrors());
            }
        } catch (Exception ex) {
            failedIds.add(raw.getId());
            log.error("转换失败: id={}", raw.getId(), ex);
        }
    }

    // 返回部分成功
    if (!failedIds.isEmpty()) {
        return ProcessResult.partialSuccess(
            dataList,
            nextCursor,
            String.format("部分数据转换失败: %d/%d",
                failedIds.size(), rawDataList.size())
        );
    }

    return ProcessResult.success(dataList, nextCursor);
}
```

## 📊 性能优化检查清单

### 优化点 1: 并行处理

```java
// ❌ 串行处理多个数据源
for (String source : sources) {
    harvest(source);
}

// ✅ 并行处理
sources.parallelStream().forEach(this::harvest);
```

### 优化点 2: 批量转换

```java

for (RawData raw : rawList) {
    T data = transform(raw);
    dataList.add(data);
}

List<T> dataList = transformBatch(rawList);
```

## 🎓 最佳实践速查

### ✅ 应该做

1. **使用 @Component 自动注册**
   - Provider 和 Processor 都添加 `@Component`
   - 依赖 Spring 自动发现和注入

2. **完善的日志记录**
   - 每个关键步骤记录日志
   - 使用结构化日志（包含 source, type, count 等）

3. **容错处理**
   - 转换失败不影响其他数据
   - 使用 `partialSuccess` 返回部分结果

4. **类型安全**
   - 始终使用 `TypeReference`
   - 避免强制类型转换

5. **配置外部化**
   - 从 `ProvenanceConfig` 读取配置
   - 不要硬编码参数

### ❌ 不应该做

1. **不要跳过类型验证**
   ```java
   // ❌ 危险
   List<Object> data = (List<Object>) result.data();

   // ✅ 安全
   TypeReference<MyData> typeRef = new TypeReference<>() {};
   DataFetchResult<MyData> result = port.fetchData(..., typeRef, ...);
   List<MyData> data = result.data();
   ```

2. **不要忽略验证**
   ```java
   // ❌ 跳过验证
   return ProcessResult.success(dataList, cursor);

   // ✅ 验证数据
   List<T> validData = dataList.stream()
       .filter(data -> validate(data).isValid())
       .collect(Collectors.toList());
   return ProcessResult.success(validData, cursor);
   ```

3. **不要吞掉异常**
   ```java
   // ❌ 吞掉异常
   try {
       process();
   } catch (Exception ex) {
       // 什么都不做
   }

   // ✅ 记录日志并返回错误
   try {
       process();
   } catch (Exception ex) {
       log.error("处理失败", ex);
       return ProcessResult.failure("处理失败: " + ex.getMessage());
   }
   ```
