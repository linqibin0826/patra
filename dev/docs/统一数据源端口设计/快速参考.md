## 核心概念

### 三层职责划分

```
Framework 层（starter-provenance）
└─ DataSourceProvider（通用参数）
   └─ preparePlan(query, params, config) → PlanMetadata

Infrastructure 层（Ingest）
└─ DataSourceAdapter（参数提取）
   └─ 提取 ExecutionContext → (query, params, config)
   └─ 调用 DataSourceProvider

Domain 层（Ingest）
└─ DataSourcePort（Ingest 接口）
   └─ preparePlan(ExecutionContext, DataType) → PlanMetadata

Application 层（Ingest）
└─ UnifiedBatchPlanner（业务逻辑）
   └─ 生成批次逻辑
```

### 统一端口

**Ingest Domain 层**：
```java
public interface DataSourcePort {
    // 计划阶段：获取元数据（接收 ExecutionContext）
    PlanMetadata preparePlan(ExecutionContext context, DataType dataType);

    // 执行阶段：获取数据
    <T> DataFetchResult<T> fetchData(...);

    // 能力查询
    boolean supports(String provenanceCode, DataType dataType);
    Set<DataType> getSupportedTypes(String provenanceCode);
}
```

**Framework 层**：
```java
public interface DataSourceProvider {
    // 准备计划（只接收通用参数）
    PlanMetadata preparePlan(String query, JsonNode params, ProvenanceConfig config);

    // 获取数据
    <T> ProviderResult<T> fetchData(ProviderRequest request, DataType dataType, Class<T> targetClass);
}
```

### 元数据继承体系（通用模型）

```
PlanMetadata（抽象基类 - 可在多模块复用）
├─ PubmedPlanMetadata（WebEnv + QueryKey）
├─ EpmcPlanMetadata（CursorMark）
└─ DoajPlanMetadata（ScrollId + PageSize）
```

### 架构分层

```
Application（Ingest 业务逻辑：批次生成）
    ↓
Domain(Port)（Ingest 接口：接收 ExecutionContext）
    ↑
Infrastructure（参数提取：ExecutionContext → 通用参数）
    ↓
Framework（DataSourceProvider：只接收通用参数）
```

## 关键设计决策

### ✅ 采纳的设计

| 决策 | 理由 | 收益 |
|------|------|------|
| **统一端口** | 计划和执行是紧密相关的操作 | 高内聚、简化依赖 |
| **继承体系** | 类型安全，编译时检查 | 避免运行时错误 |
| **保留适配器** | 提供协议转换、错误处理 | 保持 Domain 纯净 |
| **直接用框架** | starter-provenance 足够稳定 | 减少调用链路 |

### ❌ 拒绝的设计

| 方案 | 理由 |
|------|------|
| 分离的端口（PubmedSearchPort + DataSourcePort） | 职责割裂，增加复杂度 |
| 使用 Map<String, Object> 存储元数据 | 运行时类型转换，不安全 |
| 删除适配器层 | 失去协议转换价值 |

## 常见使用模式

### 模式 1：使用统一端口进行计划

```java
@Component
@RequiredArgsConstructor
public class MyPlanner {
    private final DataSourcePort dataSourcePort;

    public BatchPlan plan(ExecutionContext ctx) {
        // 1. 准备计划元数据
        PlanMetadata plan = dataSourcePort.preparePlan(ctx, DataType.LITERATURE);

        // 2. 检查结果
        if (plan.totalCount() == 0) {
            return BatchPlan.empty();
        }

        // 3. 使用多态处理不同数据源
        if (plan instanceof PubmedPlanMetadata pubmedPlan) {
            return generatePubmedBatches(pubmedPlan, ctx);
        }
        // ...
    }
}
```

### 模式 2：利用 WebEnv 优化执行

```java
@Component
@RequiredArgsConstructor
public class MyExecutor {
    private final DataSourcePort dataSourcePort;

    public void execute(ExecutionContext ctx, Batch batch) {
        // 1. 从 context 中获取 PlanMetadata
        PlanMetadata plan = ctx.planMetadata();

        // 2. 如果有 WebEnv，则优化请求
        if (plan instanceof PubmedPlanMetadata pubmedPlan && pubmedPlan.hasSessionToken()) {
            // 执行时可以使用 WebEnv 优化性能
            log.info("使用 WebEnv: {}", pubmedPlan.webEnv());
        }

        // 3. 获取数据
        DataFetchResult<CanonicalLiterature> result = dataSourcePort.fetchData(
            ctx,
            DataType.LITERATURE,
            new TypeReference<CanonicalLiterature>() {},
            batch
        );
    }
}
```

### 模式 3：类型安全的元数据访问

```java
// ✅ 正确：使用 instanceof + 模式匹配
if (plan instanceof PubmedPlanMetadata pubmedPlan) {
    String webEnv = pubmedPlan.webEnv();       // 编译时类型安全
    String queryKey = pubmedPlan.queryKey();
}

// ❌ 错误：使用 Map
Map<String, Object> metadata = plan.getMetadata();
String webEnv = (String) metadata.get("webEnv");  // 运行时类型转换
```

## 扩展指南

### 新增数据源（4 步 - 策略模式版本）

**核心原则**：新增数据源**无需修改**任何现有类，只需添加新的策略类。

#### Step 1：在 Framework 层创建 DataSourceProvider 实现

**位置**：`patra-spring-boot-starter-provenance`

```java
package com.patra.starter.provenance.newsource;

import com.fasterxml.jackson.databind.JsonNode;
import com.patra.common.model.DataType;
import com.patra.common.type.TypeReference;
import com.patra.starter.provenance.common.config.ProvenanceConfig;
import com.patra.common.model.plan.PlanMetadata;
import com.patra.starter.provenance.common.provider.DataSourceProvider;

@Component
public class NewSourceProvider implements DataSourceProvider {

    @Override
    public String getProvenanceCode() {
        return "newsource";
    }

    @Override
    public Set<DataType> getSupportedTypes() {
        return Set.of(DataType.LITERATURE);
    }

    @Override
    public boolean supports(DataType dataType) {
        return getSupportedTypes().contains(dataType);
    }

    @Override
    public PlanMetadata preparePlan(String query, JsonNode params, ProvenanceConfig config) {
        // 调用新数据源的 API（只接收通用参数）
        // 返回 NewSourcePlanMetadata
        return new NewSourcePlanMetadata(totalCount, specificToken, specificValue);
    }

    @Override
    public <T> ProviderResult<T> fetchData(
        ProviderRequest request,
        DataType dataType,
        TypeReference<T> typeRef) {  // ✅ 使用 TypeReference 而非 Class<T>
        // 实现数据获取逻辑
    }
}
```

#### Step 2：创建 PlanMetadata 子类

**位置**：`patra-common-model/src/main/java/com/patra/common/model/plan/NewSourcePlanMetadata.java`

**重要**：PlanMetadata 是通用领域模型，应放在 `patra-common-model` 模块。

```java
package com.patra.common.model.plan;

/**
 * NewSource 特定的计划元数据
 */
public class NewSourcePlanMetadata extends PlanMetadata {
    private final String specificToken;
    private final int specificValue;

    public NewSourcePlanMetadata(int totalCount, String specificToken, int specificValue) {
        super("newsource", totalCount);
        this.specificToken = specificToken;
        this.specificValue = specificValue;
    }

    @Override
    public boolean hasSessionToken() {
        return specificToken != null && !specificToken.isBlank();
    }

    public String specificToken() {
        return specificToken;
    }

    public int specificValue() {
        return specificValue;
    }
}
```

#### Step 3：创建批次生成策略（符合开闭原则）

**位置**：`patra-ingest-app/src/main/java/com/patra/ingest/app/usecase/execution/strategy/batch/NewSourceBatchGenerationStrategy.java`

**关键**：无需修改 `UnifiedBatchPlanner`，只需添加新策略类。

```java
package com.patra.ingest.app.usecase.execution.strategy.batch;

import com.patra.common.model.plan.NewSourcePlanMetadata;
import com.patra.common.model.plan.PlanMetadata;
import com.patra.ingest.domain.model.vo.batch.Batch;
import com.patra.ingest.domain.model.vo.execution.ExecutionContext;
import com.patra.ingest.domain.strategy.BatchGenerationStrategy;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Component;

import java.util.ArrayList;
import java.util.List;
import java.util.Map;

/**
 * NewSource 批次生成策略
 *
 * <p><strong>实现说明</strong>：
 * 使用 getSupportedType() 方法声明支持的类型,消除硬编码。
 */
@Component
@Slf4j
public class NewSourceBatchGenerationStrategy implements BatchGenerationStrategy {

    @Override
    public Class<? extends PlanMetadata> getSupportedType() {
        return NewSourcePlanMetadata.class;  // ✅ 策略自己声明类型
    }

    @Override
    public List<Batch> generateBatches(PlanMetadata plan, ExecutionContext ctx) {
        if (!(plan instanceof NewSourcePlanMetadata newSourcePlan)) {
            throw new IllegalArgumentException(
                "期望 NewSourcePlanMetadata，实际类型: " + plan.getClass().getName()
            );
        }

        List<Batch> batches = new ArrayList<>();
        int batchSize = ctx.provenanceConfigSnapshot().pagination().pageSizeValue();
        int totalCount = newSourcePlan.totalCount();
        int pageCount = (int) Math.ceil((double) totalCount / batchSize);

        log.debug("生成 NewSource 批次: totalCount={}, batchSize={}, pageCount={}",
                totalCount, batchSize, pageCount);

        for (int i = 0; i < pageCount; i++) {
            Map<String, Object> batchMetadata = newSourcePlan.hasSessionToken()
                    ? Map.of("specificToken", newSourcePlan.specificToken())
                    : Map.of();

            batches.add(Batch.builder()
                    .batchNo(i + 1)
                    .startOffset(i * batchSize)
                    .batchSize(batchSize)
                    .metadata(batchMetadata)
                    .build());
        }

        log.info("已生成 {} 个 NewSource 批次", batches.size());
        return batches;
    }
}
```

#### Step 4：自动注入（无需任何修改）

**✅ 完全自动**：`UnifiedBatchPlanner` 和 `DataSourceAdapter` 会自动注入所有新组件。

```java
// DataSourceAdapter：自动注入所有 DataSourceProvider
@Component
public class DataSourceAdapter implements DataSourcePort {
    public DataSourceAdapter(List<DataSourceProvider> providerList) {
        this.providerMap = providerList.stream()
            .collect(Collectors.toMap(
                DataSourceProvider::getProvenanceCode,
                Function.identity()
            ));
        // ✅ NewSourceProvider 会自动加入 Map
    }
}

// UnifiedBatchPlanner：自动注入所有 BatchGenerationStrategy
@Component
public class UnifiedBatchPlanner implements BatchPlanner {
    public UnifiedBatchPlanner(
            DataSourcePort dataSourcePort,
            List<BatchGenerationStrategy> strategies) {
        this.dataSourcePort = dataSourcePort;
        this.strategyMap = buildStrategyMap(strategies);
        // ✅ NewSourceBatchGenerationStrategy 会自动加入 Map
    }
}
```

## 故障排查

### 问题 1：PlanMetadata 为 null

**症状**：
```
NullPointerException at UnifiedBatchPlanner.plan()
```

**原因**：
- DataSourceAdapter.preparePlan() 返回 null

**解决**：
- 检查 switch 表达式是否覆盖所有数据源
- 确保每个 prepareXxxPlan() 方法都返回非 null

### 问题 2：类型转换失败

**症状**：
```
ClassCastException: PlanMetadata cannot be cast to PubmedPlanMetadata
```

**原因**：
- 使用了错误的类型强制转换

**解决**：
- 使用 `instanceof` + 模式匹配
- 不要手动强制转换

```java
// ✅ 正确
if (plan instanceof PubmedPlanMetadata pubmedPlan) {
    // 使用 pubmedPlan
}

// ❌ 错误
PubmedPlanMetadata pubmedPlan = (PubmedPlanMetadata) plan;
```

### 问题 3：WebEnv 为 null

**症状**：
```
PubmedPlanMetadata.webEnv() 返回 null
```

**原因**：
- PubMed ESearch 请求未设置 usehistory=y
- PubMed API 未返回 WebEnv

**解决**：
- 检查 ESearchRequest 是否包含 `usehistory("y")`
- 检查 PubMed API 响应是否包含 webEnv 字段
- 查看日志确认 API 调用是否成功

### 问题 4：批次数量为 0

**症状**：
```
BatchPlan.batches().size() == 0
```

**原因**：
- PlanMetadata.totalCount() == 0
- 或者批次生成逻辑错误

**解决**：
- 检查计划元数据是否正确获取
- 检查批次生成逻辑
- 添加日志跟踪

```java
log.info("PlanMetadata: totalCount={}, hasSessionToken={}",
    plan.totalCount(), plan.hasSessionToken());
```

### 问题 5：不支持的数据源异常

**症状**：
```
UnsupportedDataSourceException: 不支持的数据源: xxx
```

**原因**：
- DataSourceAdapter.preparePlan() 的 switch 中未包含该数据源

**解决**：
- 按照"扩展指南"添加新数据源支持
- 或者检查 provenanceCode 是否拼写错误

## 最佳实践

### 1. 使用多态而非 if-else

```java
// ✅ 好：使用模式匹配
if (plan instanceof PubmedPlanMetadata pubmedPlan) {
    return generatePubmedBatches(pubmedPlan, ctx);
} else if (plan instanceof EpmcPlanMetadata epmcPlan) {
    return generateEpmcBatches(epmcPlan, ctx);
}

// ❌ 差：使用字符串判断
if (plan.dataSourceType().equals("pubmed")) {
    PubmedPlanMetadata pubmedPlan = (PubmedPlanMetadata) plan;
    // ...
}
```

### 2. 优先使用继承而非 Map

```java
// ✅ 好：类型安全
public class PubmedPlanMetadata extends PlanMetadata {
    private final String webEnv;
    public String webEnv() { return webEnv; }
}

// ❌ 差：运行时类型转换
public class PlanMetadata {
    private Map<String, Object> metadata;
    public String getWebEnv() {
        return (String) metadata.get("webEnv");
    }
}
```

### 3. 添加详细日志

```java
@Override
public PlanMetadata preparePlan(ExecutionContext context, DataType dataType) {
    log.debug("准备计划元数据: provenance={}, dataType={}",
        context.provenanceCode(), dataType);

    PlanMetadata plan = // ...

    log.info("计划元数据已准备: totalCount={}, hasSessionToken={}",
        plan.totalCount(), plan.hasSessionToken());

    return plan;
}
```

### 4. 早期验证

```java
public PubmedPlanMetadata(int totalCount, String webEnv, String queryKey) {
    super("pubmed", totalCount);

    // 早期验证
    boolean hasWebEnv = webEnv != null && !webEnv.isBlank();
    boolean hasQueryKey = queryKey != null && !queryKey.isBlank();

    if (hasWebEnv != hasQueryKey) {
        throw new IllegalArgumentException("webEnv 和 queryKey 必须同时存在或同时为空");
    }

    this.webEnv = webEnv;
    this.queryKey = queryKey;
}
```

### 5. 使用不可变对象

```java
// ✅ 好：record 天然不可变
public record DataFetchResult<T>(...) { }

// ✅ 好：final 字段
public class PubmedPlanMetadata extends PlanMetadata {
    private final String webEnv;
    private final String queryKey;
}

// ❌ 差：可变字段
public class PubmedPlanMetadata extends PlanMetadata {
    private String webEnv;
    public void setWebEnv(String webEnv) { this.webEnv = webEnv; }
}
```
