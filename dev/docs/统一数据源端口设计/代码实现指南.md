## 0. Framework 层（starter-provenance）

Framework 层提供通用的数据源访问能力，不耦合任何业务模块（如 Ingest）。

### 0.1 DataSourceProvider 接口

**位置**：`patra-spring-boot-starter-provenance/src/main/java/com/patra/starter/provenance/common/provider/DataSourceProvider.java`

```java
package com.patra.starter.provenance.common.provider;

import com.fasterxml.jackson.databind.JsonNode;
import com.patra.common.model.DataType;
import com.patra.common.type.TypeReference;
import com.patra.starter.provenance.common.config.ProvenanceConfig;
import com.patra.common.model.plan.PlanMetadata;

import java.util.Set;

/**
 * 数据源提供者接口 - Framework 层通用接口
 *
 * <p>职责：提供数据源的基础访问能力，不涉及业务逻辑
 *
 * <p>设计原则：
 * <ul>
 *   <li>只接收通用参数，不接收业务特定对象（如 ExecutionContext）
 *   <li>可在多个模块复用（Ingest、Export 等）
 *   <li>返回通用的领域模型（PlanMetadata、ProviderResult）
 * </ul>
 *
 * @author Patra Framework Team
 * @since 0.2.0
 */
public interface DataSourceProvider {

    /**
     * 获取数据源代码
     *
     * @return 数据源标识（如 "pubmed"、"epmc"）
     */
    String getProvenanceCode();

    /**
     * 获取支持的数据类型
     *
     * @return 数据类型集合
     */
    Set<DataType> getSupportedTypes();

    /**
     * 判断是否支持指定数据类型
     *
     * @param dataType 数据类型
     * @return 如果支持则返回 true
     */
    boolean supports(DataType dataType);

    /**
     * 准备计划元数据
     *
     * <p>调用外部数据源 API 获取计划所需的元数据。
     *
     * <p>注意：此方法只接收通用的查询参数，不接收 Ingest 特定的 ExecutionContext
     *
     * @param query 查询字符串
     * @param params 查询参数（JSON 格式，可包含分页、排序等）
     * @param config 数据源配置
     * @return 计划元数据（使用继承体系支持不同数据源）
     * @throws ProvenanceClientException 数据源访问失败时抛出
     */
    PlanMetadata preparePlan(String query, JsonNode params, ProvenanceConfig config);

    /**
     * 从数据源获取数据
     *
     * <p>根据请求参数从外部数据源获取实际数据
     *
     * <p><strong>类型安全说明</strong>：使用 TypeReference 而非 Class，以保留泛型信息。
     * 这样可以避免 Infrastructure 层的类型转换，提升类型安全性。
     *
     * @param <T> 数据类型
     * @param request 提供者请求（包含 query、params、offset、limit 等）
     * @param dataType 数据类型标识
     * @param typeRef 类型引用（保留泛型信息）
     * @return 提供者结果
     * @throws ProvenanceClientException 数据源访问失败时抛出
     */
    <T> ProviderResult<T> fetchData(
        ProviderRequest request,
        DataType dataType,
        TypeReference<T> typeRef);
}
```

**关键点说明**：

1. **通用参数设计**：
   - `preparePlan(String query, JsonNode params, ProvenanceConfig config)`
   - 不接收 `ExecutionContext`（Ingest 特定）
   - 只接收通用的查询参数

2. **可复用性**：
   - PlanMetadata 是通用领域模型，可在多个模块使用
   - 其他模块（如 Export）也可以使用 DataSourceProvider

3. **责任边界**：
   - Framework 层：数据源访问（技术能力）
   - Infrastructure 层：参数提取和转换
   - Application 层：业务逻辑（如批次生成）

## 1. Domain 层（Ingest）

Ingest 模块的 Domain 层定义端口接口和领域模型，不依赖任何框架。

### 1.1 DataSourcePort 接口

**位置**：`patra-ingest-domain/src/main/java/com/patra/ingest/domain/port/DataSourcePort.java`

```java
package com.patra.ingest.domain.port;

import com.patra.common.model.DataType;
import com.patra.common.type.TypeReference;
import com.patra.ingest.domain.model.vo.batch.Batch;
import com.patra.ingest.domain.model.vo.execution.ExecutionContext;
import com.patra.ingest.domain.model.vo.plan.PlanMetadata;

import java.util.List;
import java.util.Map;
import java.util.Set;

/**
 * 统一的数据源端口 - 支持计划和执行两个阶段
 *
 * <p>职责：
 * <ul>
 *   <li>计划阶段：获取元数据（总数、分页信息、会话令牌）
 *   <li>执行阶段：获取实际数据
 *   <li>能力查询：支持的数据源和数据类型
 * </ul>
 *
 * <p>设计原则：
 * <ul>
 *   <li>高内聚：计划和执行是紧密相关的操作
 *   <li>单一抽象：一个端口代表完整的"数据源访问"能力
 *   <li>类型安全：使用泛型和继承体系保证类型正确性
 * </ul>
 *
 * <p>端口语义：此接口是六边形架构中的 <strong>输出端口(Output Port)</strong>，
 * 定义在 Domain 层，由基础设施层(Infrastructure)实现，确保领域逻辑与数据源技术解耦。
 *
 * @author Patra Architecture Team
 * @since 0.2.0
 */
public interface DataSourcePort {

    /**
     * 准备采集计划元数据
     *
     * <p>调用外部数据源 API 获取计划所需的元数据，如：
     * <ul>
     *   <li>总记录数
     *   <li>会话令牌（如 PubMed 的 WebEnv）
     *   <li>分页信息
     * </ul>
     *
     * <p>使用场景：在批次规划阶段调用，获取元数据后生成批次计划
     *
     * <p>实现要求：
     * <ul>
     *   <li>根据 provenanceCode 路由到对应的数据源实现
     *   <li>返回对应数据源的 PlanMetadata 子类（如 PubmedPlanMetadata）
     *   <li>捕获所有异常，转换为领域异常（DataSourceException）
     * </ul>
     *
     * @param context 执行上下文，包含查询条件和配置快照
     * @param dataType 数据类型标识（如 LITERATURE、JOURNAL）
     * @return 计划元数据（使用继承体系支持不同数据源）
     * @throws DataSourceException 数据源访问失败时抛出
     */
    PlanMetadata preparePlan(ExecutionContext context, DataType dataType);

    /**
     * 从数据源获取指定类型的数据
     *
     * <p>根据批次信息和计划元数据，从外部数据源获取实际数据
     *
     * <p>执行流程：
     * <ol>
     *   <li>从执行上下文中提取配置和查询信息
     *   <li>验证数据类型与 TypeReference 的一致性
     *   <li>根据批次定义构建数据源 API 请求
     *   <li>调用外部数据源 API 获取原始数据
     *   <li>将原始数据转换为指定类型的标准化模型
     *   <li>返回结果，包含数据列表、游标和错误信息
     * </ol>
     *
     * <p>类型安全：
     * <ul>
     *   <li>编译期类型检查：返回的 {@code List<T>} 保证类型安全</li>
     *   <li>运行时类型验证：通过 {@link TypeReference} 保持泛型信息</li>
     *   <li>类型一致性：{@code dataType} 必须与 {@code typeRef} 的类型匹配</li>
     * </ul>
     *
     * @param <T> 数据类型（通过 TypeReference 保持运行时泛型信息）
     * @param context 执行上下文（可包含 preparePlan 返回的 PlanMetadata）
     * @param dataType 数据类型标识
     * @param typeRef 类型引用
     * @param batch 批次定义
     * @return 数据获取结果
     * @throws IllegalArgumentException 如果 dataType 与 typeRef 不一致
     */
    <T> DataFetchResult<T> fetchData(
            ExecutionContext context,
            DataType dataType,
            TypeReference<T> typeRef,
            Batch batch);

    /**
     * 判断是否支持指定的数据源和数据类型组合
     *
     * @param provenanceCode 数据源代码
     * @param dataType 数据类型标识
     * @return 如果支持则返回 true
     */
    boolean supports(String provenanceCode, DataType dataType);

    /**
     * 获取指定数据源支持的所有数据类型
     *
     * @param provenanceCode 数据源代码
     * @return 支持的数据类型集合（不可变）
     */
    Set<DataType> getSupportedTypes(String provenanceCode);

    /**
     * 数据获取结果值对象
     *
     * <p>封装数据源获取操作的执行结果，包括成功状态、数据载荷、数据类型标识、分页游标、错误信息和扩展元数据。
     *
     * @param <T> 数据类型
     * @param success 是否成功获取数据
     * @param data 数据列表（泛型化，支持任意类型），不可变
     * @param dataType 数据类型标识
     * @param nextCursorToken 下一页游标令牌
     * @param errorMessage 错误或警告消息
     * @param fetchedCount 实际获取或尝试的记录数
     * @param errorType 错误类型
     * @param metadata 扩展元数据
     */
    record DataFetchResult<T>(
            boolean success,
            List<T> data,
            DataType dataType,
            String nextCursorToken,
            String errorMessage,
            int fetchedCount,
            ErrorType errorType,
            Map<String, Object> metadata) {

        // Compact 构造函数
        public DataFetchResult {
            data = data == null ? List.of() : List.copyOf(data);
            fetchedCount = Math.max(fetchedCount, data.size());
            errorType = errorType == null ? ErrorType.NONE : errorType;
            metadata = metadata == null ? Map.of() : Map.copyOf(metadata);
        }

        /**
         * 创建成功结果
         */
        public static <T> DataFetchResult<T> success(
                List<T> data, DataType dataType, String nextCursorToken) {
            return new DataFetchResult<>(
                    true, data, dataType, nextCursorToken,
                    null, data == null ? 0 : data.size(),
                    ErrorType.NONE, Map.of());
        }

        /**
         * 创建失败结果
         */
        public static <T> DataFetchResult<T> failure(
                DataType dataType, String errorMessage, ErrorType errorType) {
            return new DataFetchResult<>(
                    false, List.of(), dataType, null,
                    errorMessage, 0, errorType, Map.of());
        }

        /**
         * 错误类型枚举
         */
        public enum ErrorType {
            /** 无错误，操作完全成功 */
            NONE,
            /** 瞬时错误，调用方应遵守退避规则重试 */
            RETRIABLE,
            /** 终止性错误，调用方不应自动重试 */
            NON_RETRIABLE,
            /** 部分成功并带有警告 */
            PARTIAL_SUCCESS
        }
    }
}
```

### 1.2 BatchGenerationStrategy 接口

**位置**：`patra-ingest-domain/src/main/java/com/patra/ingest/domain/strategy/BatchGenerationStrategy.java`

**设计目的**：使用策略模式替代 instanceof 判断，使批次生成逻辑符合开闭原则（OCP）。

```java
package com.patra.ingest.domain.strategy;

import com.patra.common.model.plan.PlanMetadata;
import com.patra.ingest.domain.model.vo.batch.Batch;
import com.patra.ingest.domain.model.vo.execution.ExecutionContext;

import java.util.List;

/**
 * 批次生成策略接口
 *
 * <p>职责：定义如何根据计划元数据生成批次列表
 *
 * <p>设计原则：
 * <ul>
 *   <li>策略模式：每个数据源对应一个策略实现
 *   <li>开闭原则：新增数据源无需修改 UnifiedBatchPlanner
 *   <li>单一职责：每个策略类只处理一种数据源的批次生成
 * </ul>
 *
 * <p>使用场景：
 * <ul>
 *   <li>UnifiedBatchPlanner 根据 PlanMetadata 类型选择对应策略
 *   <li>通过 Spring 自动注入所有策略实现
 *   <li>使用 Map 进行策略路由（类似 DataSourceAdapter）
 * </ul>
 *
 * <p><strong>架构师评审改进（v4.1）</strong>：
 * <ul>
 *   <li>新增 {@link #getSupportedType()} 方法，消除硬编码类型列表
 *   <li>策略类自己声明支持的类型，完全符合开闭原则
 *   <li>新增数据源时无需修改任何现有代码
 * </ul>
 *
 * @author Patra Architecture Team
 * @since 0.2.0
 */
public interface BatchGenerationStrategy {

    /**
     * 获取策略支持的计划元数据类型
     *
     * <p>此方法允许策略类自己声明支持的类型，避免了在 UnifiedBatchPlanner 中硬编码类型列表。
     *
     * <p><strong>设计理由</strong>：
     * <ul>
     *   <li>消除硬编码：无需在 UnifiedBatchPlanner 中维护已知类型列表
     *   <li>完全符合 OCP：新增数据源零修改
     *   <li>类型安全：编译时检查类型匹配
     *   <li>自动发现：Spring 启动时自动构建策略 Map
     * </ul>
     *
     * @return 支持的 PlanMetadata 类型
     *
     * @example
     * <pre>{@code
     * @Override
     * public Class<? extends PlanMetadata> getSupportedType() {
     *     return PubmedPlanMetadata.class;
     * }
     * }</pre>
     *
     * @since 0.2.1
     */
    Class<? extends PlanMetadata> getSupportedType();

    /**
     * 根据计划元数据生成批次列表
     *
     * <p>批次生成规则由具体策略实现，可能包括：
     * <ul>
     *   <li>根据总数和批次大小计算批次数量
     *   <li>设置每个批次的起始偏移量和大小
     *   <li>附加数据源特定的元数据（如 WebEnv、cursorMark）
     * </ul>
     *
     * @param plan 计划元数据
     * @param ctx 执行上下文（包含配置信息）
     * @return 批次列表
     * @throws IllegalArgumentException 如果 plan 类型不匹配
     */
    List<Batch> generateBatches(PlanMetadata plan, ExecutionContext ctx);
}
```

**关键设计说明**：

1. **策略接口定义在 Domain 层**：
   - 遵循六边形架构，端口和策略接口都定义在 Domain 层
   - 具体实现在 Application 层

2. **类型安全**：
   - 使用 `supports(Class<?>)` 方法进行类型判断
   - 避免运行时类型转换错误

3. **与 DataSourceAdapter 一致的设计哲学**：
   - 都使用 Map 自动注入所有实现
   - 都通过类型进行策略路由

### 1.3 PlanMetadata 继承体系

**重要说明**：PlanMetadata 是**跨模块的通用领域模型**，应放在 `patra-common-model` 模块，而非 Ingest Domain 层。

#### 1.2.1 抽象基类

**位置**：`patra-common-model/src/main/java/com/patra/common/model/plan/PlanMetadata.java`

```java
package com.patra.common.model.plan;

import java.time.Instant;
import java.util.HashMap;
import java.util.Map;

/**
 * 计划元数据基类
 *
 * <p>封装将采集任务分解为批次所需的信息，同时允许执行阶段重用上游缓存。
 *
 * <p>设计原则：
 * <ul>
 *   <li>使用继承体系支持不同数据源的特定需求
 *   <li>提供抽象方法供子类实现数据源特定逻辑
 *   <li>保持类型安全，避免使用 Map<String, Object>
 * </ul>
 *
 * <p>业务约束：
 * <ul>
 *   <li>totalCount 必须 >= 0
 *   <li>dataSourceType 不能为空
 * </ul>
 *
 * @author Patra Architecture Team
 * @since 0.2.0
 */
public abstract class PlanMetadata {

    private final String dataSourceType;
    private final int totalCount;
    private final Instant plannedAt;
    private final Map<String, Object> extensionMetadata;

    protected PlanMetadata(String dataSourceType, int totalCount) {
        if (dataSourceType == null || dataSourceType.isBlank()) {
            throw new IllegalArgumentException("dataSourceType 不能为空");
        }
        if (totalCount < 0) {
            throw new IllegalArgumentException("totalCount 必须 >= 0");
        }
        this.dataSourceType = dataSourceType;
        this.totalCount = totalCount;
        this.plannedAt = Instant.now();
        this.extensionMetadata = new HashMap<>();
    }

    /**
     * 获取数据源类型
     */
    public String dataSourceType() {
        return dataSourceType;
    }

    /**
     * 获取总记录数
     */
    public int totalCount() {
        return totalCount;
    }

    /**
     * 获取计划时间
     */
    public Instant plannedAt() {
        return plannedAt;
    }

    /**
     * 获取扩展元数据（可变）
     */
    public Map<String, Object> extensionMetadata() {
        return extensionMetadata;
    }

    /**
     * 检查元数据是否包含会话令牌
     *
     * <p>会话令牌用于在执行期间重用上游缓存（如 PubMed 的 WebEnv）
     *
     * @return 如果包含会话令牌则返回 true
     */
    public abstract boolean hasSessionToken();

    /**
     * 创建表示无可用结果的空元数据
     */
    public static PlanMetadata empty(String dataSourceType) {
        return new EmptyPlanMetadata(dataSourceType);
    }

    /**
     * 空元数据实现
     */
    private static class EmptyPlanMetadata extends PlanMetadata {
        EmptyPlanMetadata(String dataSourceType) {
            super(dataSourceType, 0);
        }

        @Override
        public boolean hasSessionToken() {
            return false;
        }
    }
}
```

#### 1.2.2 PubMed 特定实现

**位置**：`patra-common-model/src/main/java/com/patra/common/model/plan/PubmedPlanMetadata.java`

```java
package com.patra.common.model.plan;

/**
 * PubMed 特定的计划元数据
 *
 * <p>包含 PubMed ESearch API 返回的特定信息：
 * <ul>
 *   <li>webEnv - History Server 会话令牌
 *   <li>queryKey - 查询键，与 webEnv 配对使用
 * </ul>
 *
 * <p>业务约束：
 * <ul>
 *   <li>webEnv 和 queryKey 必须同时存在或同时为空
 * </ul>
 *
 * @author Patra Architecture Team
 * @since 0.2.0
 */
public class PubmedPlanMetadata extends PlanMetadata {

    private final String webEnv;
    private final String queryKey;

    public PubmedPlanMetadata(int totalCount, String webEnv, String queryKey) {
        super("pubmed", totalCount);

        boolean hasWebEnv = webEnv != null && !webEnv.isBlank();
        boolean hasQueryKey = queryKey != null && !queryKey.isBlank();

        if (hasWebEnv != hasQueryKey) {
            throw new IllegalArgumentException("webEnv 和 queryKey 必须同时存在或同时为空");
        }

        this.webEnv = webEnv;
        this.queryKey = queryKey;
    }

    @Override
    public boolean hasSessionToken() {
        return webEnv != null && !webEnv.isBlank();
    }

    public String webEnv() {
        return webEnv;
    }

    public String queryKey() {
        return queryKey;
    }

    @Override
    public String toString() {
        return String.format("PubmedPlanMetadata[totalCount=%d, hasWebEnv=%b]",
                totalCount(), hasSessionToken());
    }
}
```

#### 1.2.4 DOAJ 特定实现

**位置**：`patra-common-model/src/main/java/com/patra/common/model/plan/DoajPlanMetadata.java`

```java
package com.patra.common.model.plan;

/**
 * DOAJ 特定的计划元数据
 *
 * <p>包含 DOAJ API 返回的特定信息：
 * <ul>
 *   <li>scrollId - Elasticsearch Scroll ID
 *   <li>pageSize - 每页大小
 * </ul>
 *
 * @author Patra Architecture Team
 * @since 0.2.0
 */
public class DoajPlanMetadata extends PlanMetadata {

    private final String scrollId;
    private final int pageSize;

    public DoajPlanMetadata(int totalCount, String scrollId, int pageSize) {
        super("doaj", totalCount);
        if (pageSize <= 0) {
            throw new IllegalArgumentException("pageSize 必须 > 0");
        }
        this.scrollId = scrollId;
        this.pageSize = pageSize;
    }

    @Override
    public boolean hasSessionToken() {
        return scrollId != null && !scrollId.isBlank();
    }
```

```java
/**
 * 统一的数据源适配器
 *
 * <p>实现 Ingest Domain 层的 DataSourcePort 接口，桥接 Ingest 层和 Framework 层。
 *
 * <p>职责：
 * <ul>
 *   <li>参数提取：从 ExecutionContext 提取通用参数 (query, params, config)
 *   <li>Provider 路由：根据 provenanceCode 选择对应的 DataSourceProvider
 *   <li>协议转换：Domain 模型 ↔ Framework 模型
 *   <li>错误处理：Framework 异常 → Domain 异常

 *   <li>通过 Map 自动注入所有 DataSourceProvider
 *   <li>适配器保持薄层，价值在于参数提取和协议转换
 * </ul>
 *
 * @author Patra Architecture Team
 * @since 0.2.0

@Component
@RequiredArgsConstructor
@Slf4j
public class DataSourceAdapter implements DataSourcePort {

    // 自动注入所有 DataSourceProvider（Framework 层）
    private final Map<String, DataSourceProvider> providerMap;

    /**
     * 构造函数：将 List<DataSourceProvider> 转换为 Map<String, DataSourceProvider>
     */
    public DataSourceAdapter(List<DataSourceProvider> providerList) {
        this.providerMap = providerList.stream()
            .collect(Collectors.toMap(
                DataSourceProvider::getProvenanceCode,
                Function.identity()
            ));
        log.info("已注册 {} 个 DataSourceProvider: {}",
            providerMap.size(),
            providerMap.keySet());
    }

    @Override
    public PlanMetadata preparePlan(ExecutionContext context, DataType dataType) {
        String provenanceCode = context.provenanceCode();

        log.debug("准备计划元数据: provenance={}, dataType={}", provenanceCode, dataType);

        try {
            // 1. 获取对应的 DataSourceProvider
            DataSourceProvider provider = providerMap.get(provenanceCode);
            if (provider == null) {
                throw new UnsupportedDataSourceException(
                    "不支持的数据源: " + provenanceCode);
            }

            // 2. 从 ExecutionContext 提取通用参数
            String query = context.compiledQuery();
            JsonNode params = context.compiledParams();
            ProvenanceConfig config = toProvenanceConfig(context.provenanceConfigSnapshot());

            log.debug("调用 DataSourceProvider: provenance={}, query={}",
                provenanceCode, query);

            // 3. 调用 Framework 层的 DataSourceProvider
            PlanMetadata planMetadata = provider.preparePlan(query, params, config);

            log.info("计划元数据已准备: provenance={}, totalCount={}, hasSessionToken={}",
                provenanceCode,
                planMetadata.totalCount(),
                planMetadata.hasSessionToken());

            return planMetadata;

        } catch (ProvenanceClientException ex) {
            log.error("准备计划元数据失败(客户端错误): provenance={}, dataType={}",
                    provenanceCode, dataType, ex);
            throw new DataSourceException("数据源客户端错误: " + ex.getMessage(), ex);
        } catch (Exception ex) {
            log.error("准备计划元数据失败(未知错误): provenance={}, dataType={}",
                    provenanceCode, dataType, ex);
            throw new DataSourceException("准备计划失败: " + ex.getMessage(), ex);
        }
    }

    @Override
    public <T> DataFetchResult<T> fetchData(
            ExecutionContext context,
            DataType dataType,
            TypeReference<T> typeRef,
            Batch batch) {

        String provenanceCode = context.provenanceCode();
        log.debug("获取数据: provenance={}, dataType={}, batch={}",
                provenanceCode, dataType, batch.batchNo());

        try {
            // 1. 获取对应的 Provider
            DataSourceProvider provider = providerMap.get(provenanceCode);
            if (provider == null) {
                throw new UnsupportedDataSourceException("不支持的数据源: " + provenanceCode);
            }

            // 2. 构建 ProviderRequest（可以利用 context 中的 PlanMetadata）
            ProviderRequest request = buildProviderRequest(context, batch);

            // 3. 调用 Provider
            @SuppressWarnings("unchecked")
            Class<T> targetClass = (Class<T>) typeRef.getRawType();
            ProviderResult<T> providerResult = provider.fetchData(request, dataType, targetClass);

            // 4. 转换结果
            return convertToDataFetchResult(providerResult, dataType);

        } catch (Exception ex) {
            log.error("获取数据失败: provenance={}, dataType={}",
                    provenanceCode, dataType, ex);
            return DataFetchResult.failure(
                    dataType,
                    "数据获取失败: " + ex.getMessage(),
                    DataFetchResult.ErrorType.RETRIABLE
            );
        }
    }

    /**
     * 构建 ProviderRequest，利用 PlanMetadata 优化请求
     */
    private ProviderRequest buildProviderRequest(ExecutionContext context, Batch batch) {
        ProviderRequest.Builder builder = ProviderRequest.builder()
                .query(context.compiledQuery())
                .params(context.compiledParams())
                .offset(batch.startOffset())
                .limit(batch.batchSize());

        // 利用 PlanMetadata 优化请求（如 PubMed 的 WebEnv）
        PlanMetadata planMetadata = context.planMetadata();
        if (planMetadata instanceof PubmedPlanMetadata pubmedPlan && pubmedPlan.hasSessionToken()) {
            builder.webEnv(pubmedPlan.webEnv())
                    .queryKey(pubmedPlan.queryKey());
            log.debug("使用 PubMed WebEnv 优化请求");
        }

        return builder.build();
    }

    /**
     * 转换 ProviderResult 为 DataFetchResult
     */
    private <T> DataFetchResult<T> convertToDataFetchResult(
            ProviderResult<T> providerResult,
            DataType dataType) {

        if (providerResult.isSuccess()) {
            return DataFetchResult.success(
                    providerResult.getData(),
                    dataType,
                    providerResult.getNextCursor()
            );
        } else {
            return DataFetchResult.failure(
                    dataType,
                    providerResult.getErrorMessage(),
                    providerResult.isRetriable()
                            ? DataFetchResult.ErrorType.RETRIABLE
                            : DataFetchResult.ErrorType.NON_RETRIABLE
            );
        }
    }

    @Override
    public boolean supports(String provenanceCode, DataType dataType) {
        DataSourceProvider provider = providerMap.get(provenanceCode);
        return provider != null && provider.supports(dataType);
    }

    @Override
    public Set<DataType> getSupportedTypes(String provenanceCode) {
        DataSourceProvider provider = providerMap.get(provenanceCode);
        return provider != null ? provider.getSupportedTypes() : Set.of();
    }

    /**

    private ProvenanceConfig toProvenanceConfig(ProvenanceConfigSnapshot snapshot) {
        if (snapshot == null || snapshot.provenance() == null) {

        }

        String baseUrl = snapshot.provenance().baseUrlDefault();

        try {
            return new ProvenanceConfig(
                    baseUrl.trim(), http, pagination, windowOffset,
                    batching, retry, rateLimit);
        } catch (IllegalArgumentException ex) {
            log.warn("构建 ProvenanceConfig 失败，使用默认配置", ex);
            return null;
        }
    }
```

### 3.2 UnifiedBatchPlanner 实现（策略模式版本）

**位置**：`patra-ingest-app/src/main/java/com/patra/ingest/app/usecase/execution/strategy/planner/UnifiedBatchPlanner.java`

**职责说明**：
- ✅ **调用端口获取元数据**：使用 `DataSourcePort.preparePlan()` 获取 PlanMetadata
- ✅ **委派批次生成**：使用策略模式委派给对应的 BatchGenerationStrategy
- ✅ **上下文传递**：将 PlanMetadata 附加到 ExecutionContext 供执行阶段使用
- ❌ **不调用 Framework 层**：不直接使用 DataSourceProvider
- ❌ **不包含批次生成逻辑**：委派给策略类

**架构优势**：
- ✅ **符合开闭原则（OCP）**：新增数据源无需修改 UnifiedBatchPlanner
- ✅ **单一职责（SRP）**：每个策略类只处理一种数据源
- ✅ **与 DataSourceAdapter 设计一致**：都使用 Map 自动注入

```java

import com.patra.common.enums.ProvenanceCode;
import com.patra.ingest.domain.exception.BatchPlanningException;
import com.patra.ingest.domain.model.vo.batch.Batch;
import com.patra.ingest.domain.model.vo.batch.BatchPlan;
import com.patra.ingest.domain.model.vo.execution.ExecutionContext;
import com.patra.common.model.plan.PlanMetadata;
import com.patra.ingest.domain.port.DataSourcePort;
import com.patra.ingest.domain.strategy.BatchGenerationStrategy;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Component;

import java.util.HashMap;
import java.util.List;
import java.util.Map;

/**
 * 统一的批处理规划器（支持多数据源）
 *
 * <p>职责：
 * <ul>
 *   <li>使用 DataSourcePort.preparePlan() 获取计划元数据
 *   <li>使用策略模式委派批次生成逻辑
 *   <li>将 PlanMetadata 附加到 ExecutionContext 供执行阶段使用
 * </ul>
 *
 * <p>设计原则：
 * <ul>
 *   <li>开闭原则：新增数据源无需修改此类
 *   <li>策略模式：通过 Map 自动注入所有批次生成策略
 *   <li>与 DataSourceAdapter 设计一致：都使用策略路由
 * </ul>
 *
 * <p>责任边界说明：
 * <ul>
 *   <li>Framework 层（DataSourceProvider）：只负责获取 PlanMetadata
 *   <li>Application 层策略（BatchGenerationStrategy）：负责根据 PlanMetadata 生成批次
 *   <li>Application 层编排（UnifiedBatchPlanner）：负责策略路由和上下文传递
 * </ul>
 *
 * @author Patra Architecture Team
 * @since 0.2.0
 */
@Component
@Slf4j
public class UnifiedBatchPlanner implements BatchPlanner {

    private final DataSourcePort dataSourcePort;
    private final Map<Class<? extends PlanMetadata>, BatchGenerationStrategy> strategyMap;

    /**
     * 构造函数：自动注入所有批次生成策略
     *
     * <p>Spring 会自动注入所有 BatchGenerationStrategy 实现，
     * 构造函数将其转换为以 PlanMetadata 类型为键的 Map。
     *
     * <p><strong>架构师评审改进（v4.1）</strong>：
     * <ul>
     *   <li>消除硬编码类型列表，通过 {@link BatchGenerationStrategy#getSupportedType()} 自动发现
     *   <li>完全符合开闭原则，新增数据源零修改
     * </ul>
     *
     * @param dataSourcePort 数据源端口
     * @param strategies 所有批次生成策略
     */
    public UnifiedBatchPlanner(
            DataSourcePort dataSourcePort,
            List<BatchGenerationStrategy> strategies) {
        this.dataSourcePort = dataSourcePort;
        this.strategyMap = buildStrategyMap(strategies);

        log.info("已注册 {} 个批次生成策略: {}",
                strategyMap.size(),
                strategyMap.keySet().stream()
                        .map(Class::getSimpleName)
                        .toList());
    }

    /**
     * 构建策略 Map
     *
     * <p><strong>架构师评审改进（v4.1）</strong>：
     * 直接调用策略的 getSupportedType() 方法，消除硬编码类型列表。
     */
    private Map<Class<? extends PlanMetadata>, BatchGenerationStrategy> buildStrategyMap(
            List<BatchGenerationStrategy> strategies) {
        Map<Class<? extends PlanMetadata>, BatchGenerationStrategy> map = new HashMap<>();

        for (BatchGenerationStrategy strategy : strategies) {
            Class<? extends PlanMetadata> supportedType = strategy.getSupportedType();

            if (supportedType == null) {
                log.warn("策略 {} 返回 null 的 supportedType，跳过注册",
                        strategy.getClass().getSimpleName());
                continue;
            }

            if (map.containsKey(supportedType)) {
                log.error("检测到重复的策略类型: {}，已有策略: {}，新策略: {}",
                        supportedType.getSimpleName(),
                        map.get(supportedType).getClass().getSimpleName(),
                        strategy.getClass().getSimpleName());
                throw new IllegalStateException(
                    String.format("重复的批次生成策略: %s", supportedType.getSimpleName())
                );
            }

            map.put(supportedType, strategy);
            log.debug("注册批次生成策略: {} -> {}",
                    supportedType.getSimpleName(),
                    strategy.getClass().getSimpleName());
        }

        if (map.isEmpty()) {
            log.warn("未发现任何批次生成策略，请检查 Spring 配置");
        }

        return map;
    }

    @Override
    public ProvenanceCode getProvenanceCode() {
        // 统一规划器支持所有数据源
        return null;
    }

    @Override
    public BatchPlan plan(ExecutionContext ctx) {
        try {
            // 1. 使用统一端口准备计划元数据
            PlanMetadata planMetadata = dataSourcePort.preparePlan(
                    ctx,
                    ctx.dataType()
            );

            if (planMetadata.totalCount() == 0) {
                log.info("计划结果为空: provenance={}", ctx.provenanceCode());
                return BatchPlan.empty();
            }

            log.info("计划元数据已准备: provenance={}, totalCount={}, hasSessionToken={}",
                    ctx.provenanceCode(),
                    planMetadata.totalCount(),
                    planMetadata.hasSessionToken());

            // 2. 根据 PlanMetadata 类型选择对应策略
            BatchGenerationStrategy strategy = strategyMap.get(planMetadata.getClass());
            if (strategy == null) {
                throw new IllegalStateException(
                    "未找到对应的批次生成策略: " + planMetadata.getClass().getName()
                );
            }

            // 3. 使用策略生成批次
            List<Batch> batches = strategy.generateBatches(planMetadata, ctx);

            // 4. 将 planMetadata 附加到 context 中，供执行阶段使用
            ExecutionContext enrichedContext = ctx.withPlanMetadata(planMetadata);

            log.info("批次计划已生成: provenance={}, batchCount={}, strategy={}",
                    ctx.provenanceCode(),
                    batches.size(),
                    strategy.getClass().getSimpleName());

            return new BatchPlan(batches, enrichedContext);

        } catch (Exception ex) {
            log.error("批次规划失败: provenance={}", ctx.provenanceCode(), ex);
            throw new BatchPlanningException("批次规划失败: " + ex.getMessage(), ex);
        }
    }
}
```

**与旧版本（instanceof 方式）的对比**：

```java

package com.patra.ingest.infra.config;

 * 数据源配置

     * 注入 PubMedClient

     * <p>由 patra-spring-boot-starter-provenance 自动配置
     */

    public PubMedClient pubMedClient() {
        // 如果 starter 已经提供，这里可以省略
        return null;
    }

     * 创建 DataSourceProvider Map
     *
     * <p>将所有 DataSourceProvider 注入为 Map，key 为 provenanceCode
     */
    @Bean

                        DataSourceProvider::getProvenanceCode,
                        Function.identity()
                ));
```
