## 💎 代码风格与规范

### 命名规范

- [ ] **CHK-STYLE-001**: 类名使用大驼峰命名法
  - [ ] 实体类：`Article`, `Provenance`
  - [ ] 值对象：`ArticleId`, `Email`
  - [ ] 服务类：`ArticleService`, `ProvenanceValidator`

- [ ] **CHK-STYLE-002**: 方法名使用小驼峰命名法
  - [ ] 动词开头：`createArticle()`, `validateEmail()`
  - [ ] 布尔方法：`isValid()`, `hasPermission()`

- [ ] **CHK-STYLE-003**: 常量使用全大写下划线分隔
  - [ ] `MAX_RETRY_COUNT`, `DEFAULT_TIMEOUT`

- [ ] **CHK-STYLE-004**: 包名使用全小写
  - [ ] `com.patra.ingest.domain.model`
  - [ ] `com.patra.registry.api.dto`

### 代码格式

- [ ] **CHK-STYLE-005**: 代码格式统一
  - [ ] 使用 4 空格缩进（不使用 Tab）
  - [ ] 行长度不超过 120 字符
  - [ ] 大括号使用 Java 风格（不换行）

- [ ] **CHK-STYLE-006**: 注释符合规范
  - [ ] 类级别有 JavaDoc 注释（中文）
  - [ ] 公开方法有 JavaDoc 注释（中文）
  - [ ] 复杂逻辑有行内注释说明

## 🏗️ 设计原则

### SOLID 原则

- [ ] **CHK-SOLID-001**: 单一职责原则 (SRP)
  - [ ] 每个类只有一个变化原因
  - [ ] 类的职责明确清晰
  - [ ] 避免"上帝类"

- [ ] **CHK-SOLID-002**: 开闭原则 (OCP)
  - [ ] 对扩展开放，对修改关闭
  - [ ] 使用抽象和接口设计扩展点

- [ ] **CHK-SOLID-003**: 里氏替换原则 (LSP)
  - [ ] 子类可以替换父类
  - [ ] 继承关系正确使用

- [ ] **CHK-SOLID-004**: 接口隔离原则 (ISP)
  - [ ] 接口粒度合理
  - [ ] 避免臃肿接口

- [ ] **CHK-SOLID-005**: 依赖倒置原则 (DIP)
  - [ ] 高层模块不依赖低层模块
  - [ ] 依赖抽象而非具体实现

### DRY 原则

- [ ] **CHK-DRY-001**: 避免重复代码
  - [ ] 无重复的业务逻辑
  - [ ] 提取公共方法/工具类
  - [ ] 重用 patra-common-util 和 Hutool
