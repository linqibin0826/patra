## Git 变更上下文

**当前仓库状态：**
!`git status`

**未提交的变更（diff）：**
!`git diff HEAD --stat`

**已提交未推送的提交：**
!`git log origin/main..HEAD --oneline`

**所有本地变更汇总（相对于远程 main）：**
!`git diff origin/main...HEAD --stat`

---

## 任务目标

对所有本地代码变更（包括未提交的和已提交未 push 的）执行全面的质量检查，确保：
1. ✅ 符合六边形架构 + DDD 规范
2. ✅ 代码质量达标，无反模式和安全问题
3. ✅ 相关文档已更新（README.md、package-info.java）
4. ✅ 测试代码可以编译通过

---

## 执行指令

### 🚨 强制要求：并行执行

**你必须在单个响应中使用三个 Task 工具调用来并行启动以下三个 agents：**

#### 1. 架构合规性审查
```
subagent_type: java-hexagonal-architecture
description: 架构合规性审查
prompt: |
  基于上述 Git 变更上下文，审查所有变更的代码文件，检查：

  **核心检查项：**
  - ✅ 依赖方向是否正确（domain → app → infra/adapter）
  - ✅ domain 层是否保持纯 Java（无框架依赖）
  - ✅ DDD 模式应用是否正确（Aggregate、Entity、Value Object、Repository）
  - ✅ 端口和适配器分离是否清晰
  - ✅ 跨界上下文（Bounded Context）的集成是否合理

  **输出要求：**
  - 列出所有架构问题（如果有）
  - 对每个问题提供具体的文件位置和修复建议
  - 评估整体架构健康度（优秀/良好/需改进/严重问题）
```

#### 2. 代码质量审查
```
subagent_type: java-code-reviewer
description: 代码质量审查
prompt: |
  基于上述 Git 变更上下文，审查所有变更的代码文件，检查：

  **核心检查项：**
  - ✅ 代码异味和反模式
  - ✅ 潜在的 bug 和逻辑错误
  - ✅ 安全漏洞（SQL 注入、XSS、命令注入等 OWASP Top 10）
  - ✅ 性能问题（N+1 查询、不必要的循环、资源泄漏）
  - ✅ 代码可读性和可维护性
  - ✅ 异常处理是否恰当
  - ✅ 日志记录是否充分

  **输出要求：**
  - 按严重程度分类问题（Critical/High/Medium/Low）
  - 对每个问题提供具体的代码位置和修复建议
  - 提供代码质量评分（1-10 分）
```

#### 3. 文档完整性检查
```
subagent_type: java-documentation-architect
description: 文档完整性检查
prompt: |
  基于上述 Git 变更上下文，检查文档是否与代码变更同步：

  **核心检查项：**
  - ✅ 新增/修改的模块是否有对应的 README.md 更新
  - ✅ 新增/修改的包是否有 package-info.java 说明
  - ✅ 公共 API 是否有 JavaDoc 注释
  - ✅ 架构变更是否反映在架构文档中
  - ✅ 重要的业务逻辑是否有注释说明

  **输出要求：**
  - 列出缺失或过时的文档
  - 如果发现文档缺失或不完整，直接更新相关文档
  - 提供文档覆盖率评估
```

### 测试编译检查

**在启动上述三个 agents 后，你自己需要执行：**

```bash
mvn test-compile -pl :patra-ingest-boot,:patra-registry-boot,:patra-spring-boot-starter-provenance -am
```

**检查逻辑：**
- ✅ 如果编译成功，报告"测试编译通过"
- ❌ 如果编译失败，分析错误日志，识别问题并提供修复建议（必要时修复代码）

## 注意事项

- **并行执行**：三个 agents 必须在同一个响应中启动，不要等待一个完成再启动下一个
- **完整性**：即使某个 agent 失败，也要继续执行其他检查
- **实用性**：如果发现问题，除了指出问题外，还要提供可执行的修复建议
- **自动修复**：对于文档缺失问题，documentation-architect agent 应该直接更新文档，而不仅仅是报告
