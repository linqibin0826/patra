## 1. 设计背景

### 1.1 业务需求演变

**当前阶段（v1.0）**：
- ✅ 从 PubMed 获取文献数据（CanonicalLiterature）
- ✅ 单一数据类型，硬编码接口

**下一阶段（v2.0）**：
- 🎯 从 EPMC 获取文献全文资料
- 🎯 从 DOAJ 获取期刊详细数据
- 🎯 从 Crossref 获取引用关系数据
- 🎯 从 DrugBank 获取药品临床数据

### 1.2 核心挑战

```
┌─────────────────────────────────────────────────────────────┐
│                      架构挑战矩阵                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  挑战 1: 一个数据源提供多种数据类型                          │
│  ├─ PubMed: 文献 + 引用 + 作者                              │
│  ├─ EPMC: 文献 + 全文 + 引用                                │
│  └─ Crossref: 引用 + 文献元数据                             │
│                                                             │
│  挑战 2: 数据类型完全独立，无法统一建模                      │
│  ├─ 文献数据（CanonicalLiterature）                         │
│  ├─ 期刊数据（Journal）                                     │
│  ├─ 药品数据（Drug）                                        │
│  └─ 引用数据（Citation）                                    │
│                                                             │
│  挑战 3: 处理流程一致，但数据转换各异                        │
│  ├─ 统一流程：获取 → 转换 → 验证 → 存储                    │
│  └─ 差异点：API 调用、数据解析、字段映射                    │
│                                                             │
│  挑战 4: 保持类型安全，避免运行时错误                        │
│  ├─ 编译期类型检查                                          │
│  ├─ 运行时类型验证                                          │
│  └─ IDE 智能提示                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 设计目标

| 目标 | 优先级 | 说明 |
|------|-------|------|
| **类型安全** | P0 | 编译期 + 运行时双重保障 |
| **灵活扩展** | P0 | 新增数据类型成本 < 100 行代码 |
| **一对多支持** | P0 | 一个 Provider 管理多个 Processor |
| **架构清晰** | P1 | 保持六边形架构和 DDD 原则 |
| **独立建模** | P1 | 数据类型无需强制继承 |
| **高性能** | P2 | 二维索引 O(1) 查找 |
| **易测试** | P2 | 单元测试 + 集成测试 |

### 2.2 核心组件职责

| 层次 | 组件 | 职责 | 特性 |
|------|------|------|------|
| **Domain** | `DataSourcePort` | 定义数据获取的领域契约 | 泛型方法，类型安全 |
| **Domain** | `DataType` | 数据类型标记枚举 | 包含类型元信息 |
| **Domain** | `TypeReference<T>` | 保持泛型类型信息 | 解决类型擦除 |
| **Infrastructure** | `DataSourceAdapter` | 桥接 Domain 和 Framework | 类型验证，结果转换 |
| **Infrastructure** | `ProviderRegistry` | 二维索引管理 | O(1) 查找，支持多维查询 |
| **Framework** | `DataSourceProvider` | 可插拔的数据源提供者 | 一对多支持 |
| **Framework** | `DataProcessor<T>` | 策略化的数据处理器 | 单一职责，可复用 |
| **Framework** | `ProcessorRegistry` | 处理器注册表 | 自动发现，自动注册 |

## 3. 核心设计理念

### 3.1 设计原则

```
┌─────────────────────────────────────────────────────────────┐
│                  核心设计原则（按重要性排序）                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 类型安全优先 (Type Safety First)                        │
│     ├─ 编译期：泛型方法 + TypeReference                     │
│     ├─ 运行时：DataType 枚举 + Class 验证                   │
│     └─ 开发期：IDE 智能提示 + 重构支持                      │
│                                                             │
│  2. 策略模式解耦 (Strategy Pattern Decoupling)              │
│     ├─ DataProcessor<T> 封装转换逻辑                        │
│     ├─ 一个 Provider 管理多个 Processor                     │
│     └─ Processor 可跨 Provider 复用                         │
│                                                             │
│  3. 领域独立性 (Domain Independence)                        │
│     ├─ Domain 层纯 Java，无框架依赖                         │
│     ├─ 数据模型独立，无强制继承                             │
│     └─ 可选的 CanonicalEntity 接口                          │
│                                                             │
│  4. 开闭原则 (Open-Closed Principle)                        │
│     ├─ 新增数据类型：添加 Processor                         │
│     ├─ 新增数据源：实现 Provider                            │
│     └─ 无需修改核心框架                                     │
│                                                             │
│  5. 单一职责 (Single Responsibility)                        │
│     ├─ Port: 领域契约                                       │
│     ├─ Adapter: 模型转换                                    │
│     ├─ Provider: 数据源管理                                 │
│     └─ Processor: 数据处理                                  │
│                                                             │
│  6. 依赖倒置 (Dependency Inversion)                         │
│     ├─ Domain 定义接口                                      │
│     ├─ Infrastructure 实现接口                              │
│     └─ 依赖方向：Framework → Infra → Domain                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 架构决策（ADR）

**ADR-0001: 采用策略模式而非泛型类**

- **决策**：使用 `DataProcessor<T>` 策略而非 `DataSourceProvider<T>` 泛型类
- **理由**：
  1. Spring Bean 管理更简单（无需为每个类型创建 Bean）
  2. 一个 Provider 自然支持多种类型
  3. Processor 可跨 Provider 复用
  4. 扩展成本更低（100 行 vs 200 行）
- **影响**：需要运行时类型验证，但通过 TypeReference 解决

**ADR-0002: 不强制继承统一基类**

- **决策**：提供可选的 `CanonicalEntity` 接口，而非强制继承 `CanonicalData`
- **理由**：
  1. 数据类型差异巨大（文献 vs 药品 vs 期刊）
  2. 避免继承约束，保持建模灵活性
  3. 独立的数据类型更易于理解和维护
- **影响**：需要通过策略模式处理差异，但提高了灵活性

**ADR-0003: 使用二维索引管理 Provider**

- **决策**：`Map<(provenanceCode, dataType), Provider>` 复合键索引
- **理由**：
  1. O(1) 查找性能
  2. 支持多维查询（按数据源、按类型）
  3. 清晰表达一对多关系
- **影响**：ProviderRegistry 实现稍复杂，但查询性能最优

### 3.3 与 v1.0 的对比

| 维度 | v1.0（单一类型） | v2.0（多数据类型） |
|------|-----------------|-------------------|
| **数据类型** | 仅支持 CanonicalLiterature | 支持 Literature, Journal, Drug, Citation 等 |
| **类型安全** | 硬编码 `List<CanonicalLiterature>` | 泛型方法 + TypeReference |
| **一对多** | 不支持 | 天然支持（Provider + Processor） |
| **扩展成本** | 修改接口（破坏性） | 添加 Processor（非破坏性） |
| **数据建模** | 固定结构 | 完全独立，可选接口 |
| **索引方式** | 单维（provenanceCode） | 二维（provenanceCode + dataType） |

## 4. 类型标记机制

### 4.1 DataType 枚举设计

```java
// Domain 层 - com.patra.ingest.domain.model
package com.patra.ingest.domain.model;

/**
 * 数据类型枚举 - 系统支持的所有数据类型标记
 *
 * <p>设计理念：</p>
 * <ul>
 *   <li>作为类型标记，而非业务枚举</li>
 *   <li>包含类型元信息（code, class, description）</li>
 *   <li>支持通过 code 或 class 反查</li>
 * </ul>
 *
 * @since 2.0.0
 */
public enum DataType {

    // ========== 文献相关 ==========

    /**
     * 文献数据（标准化）
     * 来源：PubMed, EPMC, Crossref
     */
    LITERATURE(
        "literature",
        "文献数据",
        CanonicalLiterature.class,
        "canonical_literature"
    ),

    /**
     * 文献全文
     * 来源：EPMC, PMC
     */
    LITERATURE_FULLTEXT(
        "literature_fulltext",
        "文献全文",
        LiteratureFulltext.class,
        "literature_fulltext"
    ),

    // ========== 期刊相关 ==========

    /**
     * 期刊数据
     * 来源：DOAJ, JCR
     */
    JOURNAL(
        "journal",
        "期刊数据",
        Journal.class,
        "canonical_journal"
    ),

    /**
     * 期刊指标（影响因子等）
     * 来源：JCR, Scopus
     */
    JOURNAL_METRICS(
        "journal_metrics",
        "期刊指标",
        JournalMetrics.class,
        "journal_metrics"
    ),

    // ========== 引用相关 ==========

    /**
     * 引用关系
     * 来源：Crossref, OpenCitations
     */
    CITATION(
        "citation",
        "引用数据",
        Citation.class,
        "canonical_citation"
    ),

    /**
     * 参考文献
     * 来源：PubMed, EPMC
     */
    REFERENCE(
        "reference",
        "参考文献",
        Reference.class,
        "literature_reference"
    ),

    // ========== 药品相关 ==========

    /**
     * 药品数据
     * 来源：DrugBank, ChEMBL
     */
    DRUG(
        "drug",
        "药品数据",
        Drug.class,
        "canonical_drug"
    ),

    /**
     * 药品相互作用
     * 来源：DrugBank
     */
    DRUG_INTERACTION(
        "drug_interaction",
        "药品相互作用",
        DrugInteraction.class,
        "drug_interaction"
    ),

    // ========== 实体相关 ==========

    /**
     * 作者信息
     * 来源：PubMed, ORCID
     */
    AUTHOR(
        "author",
        "作者信息",
        Author.class,
        "canonical_author"
    ),

    /**
     * 机构信息
     * 来源：PubMed, ROR
     */
    AFFILIATION(
        "affiliation",
        "机构信息",
        Affiliation.class,
        "canonical_affiliation"
    );

    // ========== 字段定义 ==========

    private final String code;              // 类型代码（用于配置和API）
    private final String description;       // 中文描述
    private final Class<?> dataClass;       // 对应的 Java 类
    private final String tableName;         // 数据库表名（可选）

    // ========== 构造函数 ==========

    DataType(String code, String description, Class<?> dataClass, String tableName) {
        this.code = code;
        this.description = description;
        this.dataClass = dataClass;
        this.tableName = tableName;
    }

    // ========== Getters ==========

    public String getCode() { return code; }
    public String getDescription() { return description; }
    public Class<?> getDataClass() { return dataClass; }
    public String getTableName() { return tableName; }

    // ========== 类型检查方法 ==========

    /**
     * 检查给定的类是否可以赋值给此数据类型
     */
    public boolean isAssignableFrom(Class<?> clazz) {
        return dataClass.isAssignableFrom(clazz);
    }

    /**
     * 判断是否为关系型数据类型（需要关联其他实体）
     */
    public boolean isRelational() {
        return this == CITATION
            || this == REFERENCE
            || this == DRUG_INTERACTION
            || this == LITERATURE_FULLTEXT;
    }

    /**
     * 判断是否为核心实体类型
     */
    public boolean isCoreEntity() {
        return this == LITERATURE
            || this == JOURNAL
            || this == DRUG
            || this == AUTHOR;
    }

    // ========== 查找方法 ==========

    /**
     * 通过类型代码查找
     * @throws IllegalArgumentException 如果找不到对应的类型
     */
    public static DataType fromCode(String code) {
        for (DataType type : values()) {
            if (type.code.equalsIgnoreCase(code)) {
                return type;
            }
        }
        throw new IllegalArgumentException("Unknown DataType code: " + code);
    }

    /**
     * 通过类型代码查找（返回 Optional）
     */
    public static Optional<DataType> findByCode(String code) {
        for (DataType type : values()) {
            if (type.code.equalsIgnoreCase(code)) {
                return Optional.of(type);
            }
        }
        return Optional.empty();
    }

    /**
     * 通过 Class 查找
     */
    public static DataType fromClass(Class<?> clazz) {
        for (DataType type : values()) {
            if (type.dataClass.equals(clazz)) {
                return type;
            }
        }
        throw new IllegalArgumentException("Unknown DataType for class: " + clazz.getName());
    }

    /**
     * 通过 Class 查找（返回 Optional）
     */
    public static Optional<DataType> findByClass(Class<?> clazz) {
        for (DataType type : values()) {
            if (type.dataClass.equals(clazz)) {
                return Optional.of(type);
            }
        }
        return Optional.empty();
    }

    // ========== 分组方法 ==========

    /**
     * 获取所有文献相关类型
     */
    public static Set<DataType> literatureTypes() {
        return Set.of(LITERATURE, LITERATURE_FULLTEXT, REFERENCE);
    }

    /**
     * 获取所有期刊相关类型
     */
    public static Set<DataType> journalTypes() {
        return Set.of(JOURNAL, JOURNAL_METRICS);
    }

    /**
     * 获取所有药品相关类型
     */
    public static Set<DataType> drugTypes() {
        return Set.of(DRUG, DRUG_INTERACTION);
    }
}
```

### 4.2 TypeReference 设计

```java
// Domain 层 - com.patra.ingest.domain.model
package com.patra.ingest.domain.model;

import java.lang.reflect.ParameterizedType;
import java.lang.reflect.Type;

/**
 * 类型引用 - 保持泛型类型信息（解决类型擦除问题）
 *
 * <p>借鉴 Jackson TypeReference 的设计，通过匿名内部类捕获泛型参数。</p>
 *
 * <h2>使用示例：</h2>
 * <pre>{@code
 * // 创建类型引用
 * TypeReference<CanonicalLiterature> typeRef =
 *     new TypeReference<CanonicalLiterature>() {};
 *
 * // 获取类型信息
 * Class<CanonicalLiterature> rawType = typeRef.getRawType();
 * Type type = typeRef.getType();
 * }</pre>
 *
 * @param <T> 引用的类型
 * @since 2.0.0
 */
public abstract class TypeReference<T> {

    private final Type type;
    private final Class<T> rawType;

    /**
     * 构造函数（protected，必须通过匿名内部类创建）
     */
    protected TypeReference() {
        Type superClass = getClass().getGenericSuperclass();

        if (!(superClass instanceof ParameterizedType)) {
            throw new IllegalArgumentException(
                "TypeReference 必须带泛型参数，例如: new TypeReference<CanonicalLiterature>() {}"
            );
        }

        ParameterizedType parameterized = (ParameterizedType) superClass;
        this.type = parameterized.getActualTypeArguments()[0];
        this.rawType = extractRawType(this.type);
    }

    /**
     * 获取完整的类型信息（包含泛型参数）
     */
    public Type getType() {
        return type;
    }

    /**
     * 获取原始类型（擦除泛型参数）
     */
    public Class<T> getRawType() {
        return rawType;
    }

    /**
     * 检查类型是否兼容
     */
    public boolean isAssignableFrom(Class<?> clazz) {
        return rawType.isAssignableFrom(clazz);
    }

    /**
     * 提取原始类型
     */
    @SuppressWarnings("unchecked")
    private Class<T> extractRawType(Type type) {
        if (type instanceof Class) {
            return (Class<T>) type;
        } else if (type instanceof ParameterizedType) {
            return (Class<T>) ((ParameterizedType) type).getRawType();
        }
        throw new IllegalArgumentException("无法提取原始类型: " + type);
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof TypeReference)) return false;
        TypeReference<?> that = (TypeReference<?>) o;
        return type.equals(that.type);
    }

    @Override
    public int hashCode() {
        return type.hashCode();
    }

    @Override
    public String toString() {
        return "TypeReference<" + type.getTypeName() + ">";
    }
}
```

```java
// Framework 层 - com.patra.starter.provenance.common.processor
package com.patra.starter.provenance.common.processor;

/**
 * 数据处理器策略接口 - 封装数据获取、转换、验证逻辑
 *
 * <p>设计理念：</p>
 * <ul>
 *   <li>单一职责：一个 Processor 只处理一种数据类型</li>
 *   <li>可复用：Processor 可以被多个 Provider 使用</li>
 *   <li>可测试：独立的处理逻辑易于单元测试</li>
 * </ul>
 *
 * <h2>实现指南：</h2>
 * <ol>
 *   <li>实现 {@link #getDataType()} 返回处理的数据类型</li>
 *   <li>实现 {@link #process(ProviderRequest, ProviderContext)} 完成数据获取</li>
 *   <li>实现 {@link #transform(Object)} 完成数据转换</li>
 *   <li>实现 {@link #validate(Object)} 完成数据验证</li>
 *   <li>添加 @Component 注解，自动注册</li>
 * </ol>
 *
 * @param <T> 处理的数据类型
 * @since 2.0.0
 */
public interface DataProcessor<T> {

    /**
     * 获取处理器对应的数据类型
     */
    DataType getDataType();

    /**
     * 判断是否支持指定的数据类型
     */
    default boolean supports(DataType dataType) {
        return getDataType().equals(dataType);
    }

    /**
     * 处理数据获取请求（核心方法）
     *
     * <p>执行流程：</p>
     * <ol>
     *   <li>调用外部 API 获取原始数据</li>
     *   <li>调用 {@link #transform(Object)} 转换数据</li>
     *   <li>调用 {@link #validate(Object)} 验证数据</li>
     *   <li>构建 ProcessResult 返回</li>
     * </ol>
     *
     * @param request 提供者请求
     * @param context 提供者上下文（包含配置、客户端等）
     * @return 处理结果
     */
    ProcessResult<T> process(ProviderRequest request, ProviderContext context);

    /**
     * 验证数据有效性
     *
     * @param data 待验证的数据
     * @return 验证结果
     */
    ValidationResult validate(T data);

    /**
     * 转换原始数据为目标类型
     *
     * @param rawData 原始数据（来自外部 API）
     * @return 转换后的数据
     * @throws TransformationException 如果转换失败
     */
    T transform(Object rawData) throws TransformationException;

    // ========== 内部类：处理结果 ==========

    /**
     * 处理结果
     */
    @Builder
    record ProcessResult<T>(
        boolean success,
        List<T> data,
        String nextCursor,
        String errorMessage,
        ProcessStatus status,
        Map<String, Object> metadata
    ) {
        /**
         * 处理状态
         */
        public enum ProcessStatus {
            /** 完全成功 */
            SUCCESS,
            /** 部分成功（部分数据转换失败） */
            PARTIAL_SUCCESS,
            /** 失败 */
            FAILED,
            /** 验证错误 */
            VALIDATION_ERROR
        }

        /**
         * 成功结果
         */
        public static <T> ProcessResult<T> success(List<T> data, String nextCursor) {
            return ProcessResult.<T>builder()
                .success(true)
                .data(data)
                .nextCursor(nextCursor)
                .status(ProcessStatus.SUCCESS)
                .build();
        }

        /**
         * 部分成功结果
         */
        public static <T> ProcessResult<T> partialSuccess(
                List<T> data, String nextCursor, String warningMessage) {
            return ProcessResult.<T>builder()
                .success(true)
                .data(data)
                .nextCursor(nextCursor)
                .errorMessage(warningMessage)
                .status(ProcessStatus.PARTIAL_SUCCESS)
                .build();
        }

        /**
         * 失败结果
         */
        public static <T> ProcessResult<T> failure(String errorMessage) {
            return ProcessResult.<T>builder()
                .success(false)
                .data(List.of())
                .errorMessage(errorMessage)
                .status(ProcessStatus.FAILED)
                .build();
        }
    }

    // ========== 内部类：验证结果 ==========

    /**
     * 验证结果
     */
    @Value
    class ValidationResult {
        boolean valid;
        List<String> errors;

        public static ValidationResult success() {
            return new ValidationResult(true, List.of());
        }

        public static ValidationResult failure(String... errors) {
            return new ValidationResult(false, Arrays.asList(errors));
        }

        public static ValidationResult failure(List<String> errors) {
            return new ValidationResult(false, errors);
        }
    }
}
```

```java
        }
```

### 5.3 ProcessorRegistry 设计

```java
// Framework 层 - com.patra.starter.provenance.common.processor
package com.patra.starter.provenance.common.processor;

/**
 * 处理器注册表 - 管理所有 DataProcessor
 *
 * <p>设计理念：</p>
 * <ul>
 *   <li>自动发现：通过 Spring 自动注入所有 DataProcessor</li>
 *   <li>自动注册：按 DataType 索引</li>
 *   <li>类型安全：通过泛型保证类型安全</li>
 * </ul>
 *
 * @since 2.0.0
 */
@Slf4j
@Component
public class ProcessorRegistry {

    /**
     * 处理器索引：DataType -> DataProcessor
     */
    private final Map<DataType, DataProcessor<?>> processors = new ConcurrentHashMap<>();

    /**
     * 构造函数（Spring 自动注入所有 DataProcessor）
     */
    public ProcessorRegistry(List<DataProcessor<?>> discoveredProcessors) {
        if (discoveredProcessors != null) {
            discoveredProcessors.forEach(this::register);
        }
        logRegistrationSummary();
    }

    /**
     * 注册处理器
     */
    private void register(DataProcessor<?> processor) {
        if (processor == null) {
            return;
        }

        DataType dataType = processor.getDataType();

        if (processors.containsKey(dataType)) {
            log.warn("覆盖已存在的 Processor: type={}, old={}, new={}",
                dataType,
                processors.get(dataType).getClass().getSimpleName(),
                processor.getClass().getSimpleName()
            );
        }

        processors.put(dataType, processor);
        log.info("注册 DataProcessor: type={}, class={}",
            dataType, processor.getClass().getSimpleName());
    }

    /**
     * 获取处理器
     */
    @SuppressWarnings("unchecked")
    public <T> DataProcessor<T> getProcessor(DataType dataType) {
        DataProcessor<?> processor = processors.get(dataType);
        if (processor == null) {
            throw new ProcessorNotFoundException(
                "未找到 DataProcessor: type=" + dataType
            );
        }
        return (DataProcessor<T>) processor;
    }

    /**
     * 检查是否支持指定类型
     */
    public boolean supports(DataType dataType) {
        return processors.containsKey(dataType);
    }

    /**
     * 获取所有支持的类型
     */
    public Set<DataType> getSupportedTypes() {
        return Collections.unmodifiableSet(processors.keySet());
    }

    /**
     * 获取注册统计
     */
    public Map<DataType, String> getRegistryInfo() {
        return processors.entrySet().stream()
            .collect(Collectors.toMap(
                Map.Entry::getKey,
                e -> e.getValue().getClass().getSimpleName()
            ));
    }

    private void logRegistrationSummary() {
        log.info("DataProcessor 注册完成: 共 {} 个", processors.size());
        processors.forEach((type, processor) -> {
            log.debug("  - {}: {}", type, processor.getClass().getName());
        });
    }
}
```

```java
// Domain 层 - com.patra.ingest.domain.port
package com.patra.ingest.domain.port;

/**
 * 数据源端口 - 支持多种数据类型的领域契约
 *
 * <p>这是六边形架构的 Output Port，定义在 Domain 层，由 Infrastructure 层实现。</p>
 *
 * <h2>设计理念：</h2>
 * <ul>
 *   <li>泛型方法：支持多种数据类型，保持类型安全</li>
 *   <li>类型标记：通过 DataType 枚举标识数据类型</li>
 *   <li>类型引用：通过 TypeReference 保持泛型信息</li>
 *   <li>能力查询：支持查询数据源支持的类型</li>
 * </ul>
 *
 * @since 2.0.0
 */
public interface DataSourcePort {

    /**
     * 从数据源获取指定类型的数据（核心方法）
     *
     * <p>使用示例：</p>
     * <pre>{@code
     * // 获取文献数据
     * TypeReference<CanonicalLiterature> typeRef = new TypeReference<>() {};
     * DataFetchResult<CanonicalLiterature> result = port.fetchData(
     *     context,
     *     DataType.LITERATURE,
     *     typeRef,
     *     batch
     * );
     *
     * // 类型安全的数据访问
     * List<CanonicalLiterature> literatures = result.data();
     * }</pre>
     *
     * @param <T> 数据类型（由 TypeReference 指定）
     * @param context 执行上下文（包含数据源、配置快照等）
     * @param dataType 数据类型标记
     * @param typeRef 类型引用（保持泛型信息）
     * @param batch 批次定义（包含分页、游标等）
     * @return 类型安全的数据获取结果
     * @throws IllegalArgumentException 如果类型不一致
     */
    <T> DataFetchResult<T> fetchData(
        ExecutionContext context,
        DataType dataType,
        TypeReference<T> typeRef,
        Batch batch
    );

    /**
     * 检查数据源是否支持指定的数据类型
     *
     * @param provenanceCode 数据源代码（如 "pubmed", "doaj"）
     * @param dataType 数据类型
     * @return true 如果支持
     */
    boolean supports(String provenanceCode, DataType dataType);

    /**
     * 获取数据源支持的所有数据类型
     *
     * @param provenanceCode 数据源代码
     * @return 支持的数据类型集合
     */
    Set<DataType> getSupportedTypes(String provenanceCode);

    // ========== 内部类：数据获取结果 ==========

    /**
     * 数据获取结果（泛型）
     *
     * @param <T> 数据类型
     */
    @Builder
    record DataFetchResult<T>(
        boolean success,
        List<T> data,                   // 类型安全的数据列表
        DataType dataType,               // 数据类型标记
        String nextCursorToken,          // 下一页游标
        String errorMessage,             // 错误消息
        int fetchedCount,                // 实际获取数量
        ErrorType errorType,             // 错误类型
        Map<String, Object> metadata     // 额外元数据（可选）
    ) {
        /**
         * 成功结果
         */
        public static <T> DataFetchResult<T> success(
                List<T> data, DataType dataType, String nextCursorToken) {
            return DataFetchResult.<T>builder()
                .success(true)
                .data(data)
                .dataType(dataType)
                .nextCursorToken(nextCursorToken)
                .fetchedCount(data.size())
                .errorType(ErrorType.NONE)
                .build();
        }

        /**
         * 失败结果
         */
        public static <T> DataFetchResult<T> failure(
                DataType dataType, String errorMessage, ErrorType errorType) {
            return DataFetchResult.<T>builder()
                .success(false)
                .data(List.of())
                .dataType(dataType)
                .errorMessage(errorMessage)
                .errorType(errorType)
                .fetchedCount(0)
                .build();
        }

        /**
         * 部分成功结果
         */
        public static <T> DataFetchResult<T> partialSuccess(
                List<T> data, DataType dataType, String nextCursor, String warningMessage) {
            return DataFetchResult.<T>builder()
                .success(true)
                .data(data)
                .dataType(dataType)
                .nextCursorToken(nextCursor)
                .errorMessage(warningMessage)
                .fetchedCount(data.size())
                .errorType(ErrorType.PARTIAL_SUCCESS)
                .build();
        }
    }

    /**
     * 错误类型枚举
     */
    enum ErrorType {
        /** 无错误 */
        NONE,
        /** 可重试错误（网络超时、限流等） */
        RETRIABLE,
        /** 不可重试错误（认证失败、参数错误等） */
        NON_RETRIABLE,
        /** 部分成功（部分数据转换失败） */
        PARTIAL_SUCCESS
    }
}
```

### 6.2 DataSourceAdapter 适配器实现

```java
// Infrastructure 层 - com.patra.ingest.infrastructure.adapter
package com.patra.ingest.infrastructure.adapter;

/**
 * 数据源适配器 - 桥接 Domain 层和 Framework 层
 *
 * <p>职责：</p>
 * <ul>
 *   <li>实现 DataSourcePort 接口</li>
 *   <li>类型一致性验证</li>
 *   <li>Provider 解析和调用</li>
 *   <li>结果转换</li>
 * </ul>
 *
 * @since 2.0.0
 */
@Component
@RequiredArgsConstructor
@Slf4j
public class DataSourceAdapter implements DataSourcePort {

    private final ProviderRegistry providerRegistry;

    @Override
    public <T> DataFetchResult<T> fetchData(
            ExecutionContext context,
            DataType dataType,
            TypeReference<T> typeRef,
            Batch batch) {

        long startTime = System.currentTimeMillis();
        String provenanceCode = context.provenanceCode();

        try {
            // 步骤 1: 验证类型一致性
            validateTypeConsistency(dataType, typeRef);
            log.debug("类型验证通过: dataType={}, typeRef={}",
                dataType, typeRef.getRawType().getSimpleName());

            // 步骤 2: 解析 Provider
            DataSourceProvider provider = providerRegistry
                .getProvider(provenanceCode, dataType)
                .orElseThrow(() -> new ProviderNotFoundException(
                    String.format("未找到 Provider: source=%s, type=%s",
                        provenanceCode, dataType)
                ));

            log.debug("找到 Provider: source={}, type={}, provider={}",
                provenanceCode, dataType, provider.getClass().getSimpleName());

            // 步骤 3: 构建 Provider 请求
            ProviderRequest request = buildProviderRequest(context, batch);

            // 步骤 4: 调用 Provider
            ProviderResult<T> providerResult = provider.fetchData(
                request,
                dataType,
                typeRef.getRawType()
            );

            // 步骤 5: 转换结果
            DataFetchResult<T> result = convertToDataFetchResult(providerResult);

            long duration = System.currentTimeMillis() - startTime;
            log.info("数据获取完成: source={}, type={}, success={}, count={}, duration={}ms",
                provenanceCode, dataType, result.success(), result.fetchedCount(), duration);

            return result;

        } catch (ProviderNotFoundException ex) {
            log.error("Provider 未找到: source={}, type={}", provenanceCode, dataType, ex);
            return DataFetchResult.failure(
                dataType,
                ex.getMessage(),
                ErrorType.NON_RETRIABLE
            );
        } catch (TypeMismatchException ex) {
            log.error("类型不匹配: source={}, type={}", provenanceCode, dataType, ex);
            return DataFetchResult.failure(
                dataType,
                ex.getMessage(),
                ErrorType.NON_RETRIABLE
            );
        } catch (Exception ex) {
            log.error("数据获取失败: source={}, type={}", provenanceCode, dataType, ex);
            return DataFetchResult.failure(
                dataType,
                "系统错误: " + ex.getMessage(),
                ErrorType.RETRIABLE
            );
        }
    }

    @Override
    public boolean supports(String provenanceCode, DataType dataType) {
        return providerRegistry.supports(provenanceCode, dataType);
    }

    @Override
    public Set<DataType> getSupportedTypes(String provenanceCode) {
        return providerRegistry.getSupportedTypes(provenanceCode);
    }

    /**
     * 验证类型一致性
     */
    private <T> void validateTypeConsistency(DataType dataType, TypeReference<T> typeRef) {
        Class<?> expectedClass = dataType.getDataClass();
        Class<?> actualClass = typeRef.getRawType();

        // 处理 List 类型
        if (List.class.isAssignableFrom(actualClass)) {
            Type type = typeRef.getType();
            if (type instanceof ParameterizedType) {
                Type[] args = ((ParameterizedType) type).getActualTypeArguments();
                if (args.length > 0 && args[0] instanceof Class) {
                    actualClass = (Class<?>) args[0];
                }
            }
        }

        if (!expectedClass.isAssignableFrom(actualClass)) {
            throw new TypeMismatchException(
                String.format("类型不匹配: DataType 期望 %s, TypeReference 提供 %s",
                    expectedClass.getName(), actualClass.getName())
            );
        }
    }

    /**
     * 构建 Provider 请求
     */
    private ProviderRequest buildProviderRequest(ExecutionContext context, Batch batch) {
        // 转换配置快照为运行时配置
        ProvenanceConfig runtimeConfig = convertConfig(context.configSnapshot());

        // 合并参数（context.compiledParams + batch.params）
        JsonNode mergedParams = mergeParams(context.compiledParams(), batch.params());

        // 确定查询字符串优先级（batch.query > context.compiledQuery）
        String query = StringUtils.hasText(batch.query())
            ? batch.query()
            : context.compiledQuery();

        // 构建批次执行参数
        BatchExecutionParams executionParams = new BatchExecutionParams(query, mergedParams);

        // 构建批次元数据
        BatchMetadata metadata = BatchMetadata.builder()
            .batchNo(batch.batchNo())
            .cursorToken(batch.cursorToken())
            .pageSize(batch.pageSize())
            .build();

        // 构建请求
        return ProviderRequest.builder()
            .operationCode(context.operationCode())
            .config(runtimeConfig)
            .executionParams(executionParams)
            .metadata(metadata)
            .build();
    }

    /**
     * 转换 Provider 结果为 Domain 结果
     */
    private <T> DataFetchResult<T> convertToDataFetchResult(ProviderResult<T> providerResult) {
        return DataFetchResult.<T>builder()
            .success(providerResult.success())
            .data(providerResult.data())
            .dataType(providerResult.dataType())
            .nextCursorToken(providerResult.nextCursorToken())
            .errorMessage(providerResult.errorMessage())
            .fetchedCount(providerResult.fetchedCount())
            .errorType(convertErrorType(providerResult.errorType()))
            .metadata(providerResult.metadata())
            .build();
    }

    /**
     * 转换错误类型
     */
    private ErrorType convertErrorType(ProviderResult.ErrorType providerErrorType) {
        return switch (providerErrorType) {
            case NONE -> ErrorType.NONE;
            case RETRIABLE -> ErrorType.RETRIABLE;
            case NON_RETRIABLE -> ErrorType.NON_RETRIABLE;
            case PARTIAL_SUCCESS -> ErrorType.PARTIAL_SUCCESS;
        };
    }

    // 其他辅助方法（convertConfig, mergeParams）...
}
```

```java
     * 获取指定类型的数据（泛型方法）

     * @param <T> 数据类型
     * @param request 提供者请求
     * @param dataType 数据类型
     * @param targetClass 目标类
     * @return 提供者结果
     */
    <T> ProviderResult<T> fetchData(
        ProviderRequest request,
        DataType dataType,
        Class<T> targetClass
    );

    /**
     * 获取数据处理器（可选实现）
     */
    default Optional<DataProcessor<?>> getProcessor(DataType dataType) {
        return Optional.empty();
    }

    // ========== 内部类：提供者结果 ==========

    @Builder
    record ProviderResult<T>(
        boolean success,
        List<T> data,
        DataType dataType,
        String nextCursorToken,
        String errorMessage,
        int fetchedCount,
        ErrorType errorType,
        Map<String, Object> metadata
    ) {
        enum ErrorType {
            NONE, RETRIABLE, NON_RETRIABLE, PARTIAL_SUCCESS
        }

        public static <T> ProviderResult<T> success(List<T> data, DataType dataType, String nextCursor) {
            return ProviderResult.<T>builder()
                .success(true)
                .data(data)
                .dataType(dataType)
                .nextCursorToken(nextCursor)
                .fetchedCount(data.size())
                .errorType(ErrorType.NONE)
                .build();
        }

        public static <T> ProviderResult<T> failure(DataType dataType, String errorMessage, ErrorType errorType) {
            return ProviderResult.<T>builder()
                .success(false)
                .data(List.of())
                .dataType(dataType)
                .errorMessage(errorMessage)
                .errorType(errorType)
                .build();
        }
    }
```

### 7.2 ProviderRegistry 二维索引

```java
// Framework 层 - com.patra.starter.provenance.common.registry
package com.patra.starter.provenance.common.registry;

/**
 * 提供者注册表 - 二维索引管理
 *
 * <p>索引策略：</p>
 * <ul>
 *   <li>主索引：Map<ProviderKey, DataSourceProvider></li>
 *   <li>ProviderKey = (provenanceCode, dataType)</li>
 *   <li>辅助索引 1：按 provenanceCode 查询</li>
 *   <li>辅助索引 2：按 dataType 查询</li>
 * </ul>
 *
 * @since 2.0.0
 */
@Slf4j
@Component
public class ProviderRegistry {

    // ========== 主索引：(provenanceCode, dataType) -> Provider ==========
    private final Map<ProviderKey, DataSourceProvider> providersByType = new ConcurrentHashMap<>();

    // ========== 辅助索引 1：provenanceCode -> Set<DataType> ==========
    private final Map<String, Set<DataType>> typesByProvenance = new ConcurrentHashMap<>();

    // ========== 辅助索引 2：dataType -> List<Provider> ==========
    private final Map<DataType, List<DataSourceProvider>> providersByDataType = new ConcurrentHashMap<>();

    /**
     * 复合键：数据源代码 + 数据类型
     */
    @Value
    private static class ProviderKey {
        String provenanceCode;
        DataType dataType;
    }

    /**
     * 构造函数（Spring 自动注入所有 DataSourceProvider）
     */
    public ProviderRegistry(List<DataSourceProvider> discoveredProviders) {
        if (discoveredProviders != null) {
            discoveredProviders.forEach(this::register);
        }
        logRegistrationSummary();
    }

    /**
     * 注册 Provider
     */
    private void register(DataSourceProvider provider) {
        if (provider == null) {
            return;
        }

        String code = normalize(provider.getProvenanceCode());

        // 1. 注册到主索引
        for (DataType dataType : provider.getSupportedDataTypes()) {
            ProviderKey key = new ProviderKey(code, dataType);

            if (providersByType.containsKey(key)) {
                log.warn("覆盖已存在的 Provider: key={}, old={}, new={}",
                    key,
                    providersByType.get(key).getClass().getSimpleName(),
                    provider.getClass().getSimpleName()
                );
            }

            providersByType.put(key, provider);
        }

        // 2. 注册到辅助索引 1：按数据源
        typesByProvenance.put(code, Set.copyOf(provider.getSupportedDataTypes()));

        // 3. 注册到辅助索引 2：按数据类型
        for (DataType dataType : provider.getSupportedDataTypes()) {
            providersByDataType.computeIfAbsent(
                dataType,
                k -> new CopyOnWriteArrayList<>()
            ).add(provider);
        }

        log.info("注册 Provider: code={}, types={}, class={}",
            code, provider.getSupportedDataTypes(), provider.getClass().getSimpleName());
    }

    /**
     * 根据 (provenanceCode, dataType) 获取 Provider
     */
    public Optional<DataSourceProvider> getProvider(String provenanceCode, DataType dataType) {
        ProviderKey key = new ProviderKey(normalize(provenanceCode), dataType);
        return Optional.ofNullable(providersByType.get(key));
    }

    /**
     * 获取数据源支持的所有数据类型
     */
    public Set<DataType> getSupportedTypes(String provenanceCode) {
        return typesByProvenance.getOrDefault(normalize(provenanceCode), Set.of());
    }

    /**
     * 获取支持指定数据类型的所有数据源
     */
    public List<DataSourceProvider> getProvidersByDataType(DataType dataType) {
        return providersByDataType.getOrDefault(dataType, List.of());
    }

    /**
     * 检查是否支持指定的组合
     */
    public boolean supports(String provenanceCode, DataType dataType) {
        ProviderKey key = new ProviderKey(normalize(provenanceCode), dataType);
        return providersByType.containsKey(key);
    }

    /**
     * 获取注册统计信息
     */
    public RegistryStatistics getStatistics() {
        Map<DataType, Long> typeCount = providersByDataType.entrySet().stream()
            .collect(Collectors.toMap(
                Map.Entry::getKey,
                e -> (long) e.getValue().size()
            ));

        return RegistryStatistics.builder()
            .totalProviders((long) typesByProvenance.size())
            .totalMappings((long) providersByType.size())
            .typeDistribution(typeCount)
            .build();
    }

    /**
     * 规范化数据源代码
     */
    private String normalize(String code) {
        return code == null ? "" : code.trim().toLowerCase(Locale.ROOT);
    }

    /**
     * 日志输出注册摘要
     */
    private void logRegistrationSummary() {
        log.info("Provider 注册完成:");
        log.info("  - 数据源数量: {}", typesByProvenance.size());
        log.info("  - 类型映射数: {}", providersByType.size());

        providersByDataType.forEach((type, providers) -> {
            log.info("  - {}: {} 个 Provider [{}]",
                type,
                providers.size(),
                providers.stream()
                    .map(p -> p.getClass().getSimpleName())
                    .collect(Collectors.joining(", "))
            );
        });
    }

    /**
     * 注册统计信息
     */
    @Builder
    @Value
    public static class RegistryStatistics {
        Long totalProviders;
        Long totalMappings;
        Map<DataType, Long> typeDistribution;
    }
}
```
