## 🚀 5步快速指南

### 1️⃣ 实现DataSourceProvider

**文件**: `patra-spring-boot-starter-provenance/src/main/java/com/patra/starter/provenance/{datasource}/`

```java
@Slf4j
@RequiredArgsConstructor
public class YourDataSourceProvider implements DataSourceProvider {
    
    private static final String PROVENANCE_CODE = "your_source";  // ← 核心标识
    
    private final YourApiClient client;
    private final YourArticleConverter converter;
    private final ProvenanceProperties properties;
    
    @Override
    public String getProvenanceCode() {
        return PROVENANCE_CODE;
    }
    
    @Override
    public ProviderResult fetchData(ProviderRequest request) {
        long start = System.currentTimeMillis();
        log.info("Provider start: operation={}", request.operationCode());
        
        try {
            // 步骤1: 配置合并
            ProvenanceConfig config = properties.mergeWithRuntime(
                PROVENANCE_CODE, request.config());
            
            // 步骤2: API调用
            YourResponse response = client.fetch(
                request.executionParams().query(),
                request.executionParams().params(),
                config);
            
            if (response.isEmpty()) {
                return ProviderResult.success(List.of(), null);
            }
            
            // 步骤3: 数据转换（容错处理）
            List<CanonicalLiterature> literatures = new ArrayList<>();
            List<String> failures = new ArrayList<>();
            
            for (YourRecord record : response.records()) {
                try {
                    CanonicalLiterature lit = converter.toCanonicalLiterature(record);
                    if (lit != null) literatures.add(lit);
                } catch (Exception ex) {
                    failures.add(record.id());
                    log.error("Conversion failed: id={}", record.id(), ex);
                }
            }
            
            // 步骤4: 返回结果
            if (failures.isEmpty()) {
                return ProviderResult.success(literatures, response.nextCursor());
            } else {
                return ProviderResult.partialSuccess(
                    literatures,
                    response.nextCursor(),
                    "Conversion failed for " + failures.size() + " records",
                    response.totalAttempted());
            }
            
        } catch (YourApiException ex) {
            // 错误分类示例
            if (ex.getStatusCode() == 401 || ex.getStatusCode() == 403) {
                return ProviderResult.nonRetriableFailure("Authentication failed");
            }
            if (ex.getStatusCode() == 429 || ex.getStatusCode() >= 500) {
                return ProviderResult.retriableFailure("Service unavailable");
            }
            return ProviderResult.nonRetriableFailure("API error: " + ex.getMessage());
            
        } catch (Exception ex) {
            log.error("Unexpected error", ex);
            return ProviderResult.retriableFailure("Unexpected error");
        } finally {
            log.info("Provider end: duration={}ms", System.currentTimeMillis() - start);
        }
    }
}
```

### 2️⃣ 注册为Spring Bean

**文件**: 同目录下创建Configuration类

```java
@Configuration
public class YourSourceProviderConfiguration {
    
    @Bean
    public YourApiClient yourApiClient() {
        return new YourApiClient();
    }
    
    @Bean
    public YourArticleConverter yourArticleConverter() {
        return new YourArticleConverter();
    }
    
    @Bean
    public YourDataSourceProvider yourDataSourceProvider(
        YourApiClient client,
        YourArticleConverter converter,
        ProvenanceProperties properties) {
        return new YourDataSourceProvider(client, converter, properties);
    }
}
```

### 3️⃣ 自动注册（无需额外代码）

ProviderRegistry会通过Spring自动注入发现所有DataSourceProvider实现，自动注册到内部映射表。

### 4️⃣ 配置（可选）

**文件**: `application.yml` 或 `application-your_source.yml`

```yaml
patra:
  provenance:
    datasources:
      your_source:
        http:
          timeout-connect-millis: 10000
          timeout-read-millis: 45000
        pagination:
          page-size: 100
          max-pages-per-execution: 10
        retry:
          max-retry-times: 3
```

## 📊 关键概念速查

### ErrorType 错误分类

| 类型 | 何时使用 | 上层处理 |
|------|--------|---------|
| **NONE** | 完全成功 | 继续处理 |
| **RETRIABLE** | 网络超时、HTTP 429/5xx、限流 | 触发重试 |
| **NON_RETRIABLE** | HTTP 401/403/404、参数错误 | 标记失败 |
| **PARTIAL_SUCCESS** | 部分记录转换失败 | 继续处理 + 记录警告 |

### 配置优先级

```
1. ExecutionContext.configSnapshot (最高)
   └─ 保证任务级别配置一致

2. ProvenanceProperties.datasources["your_source"]
   └─ 数据源特定覆盖

3. ProvenanceProperties.defaults (最低)
   └─ 全局基线配置
```

### 参数合并

```
ExecutionContext.compiledParams (基础参数)
    ↓ 合并
Batch.params (批次特定参数)
    ↓ 结果
完整的参数 (Batch覆盖同名字段)
```

## 📋 常见问题

### Q: 如何处理超时?
A: 异常链中检查是否包含`SocketTimeoutException`或`HttpTimeoutException`，返回`RETRIABLE`。

### Q: 如何处理限流?
A: HTTP 429状态码或特定限流异常，返回`RETRIABLE`。

### Q: 如何处理认证失败?
A: HTTP 401/403状态码，返回`NON_RETRIABLE`。

### Q: 如何处理部分失败?
A: 转换阶段记录失败项，继续处理成功项，返回`PARTIAL_SUCCESS`。

### Q: 如何自定义配置?
A: 在`application.yml`中配置`patra.provenance.datasources.{code}`。

### Q: Provider如何自动注册?
A: Spring通过`@Bean`自动发现，ProviderRegistry通过构造器注入列表并注册。

```java
catch (SpecificApiException ex) {
    // 根据业务异常分类
    return ex.isRetriable()
        ? ProviderResult.retriableFailure(msg)
        : ProviderResult.nonRetriableFailure(msg);
}

catch (InterruptedException ex) {
    Thread.currentThread().interrupt();
    return ProviderResult.retriableFailure("Interrupted");
}

catch (Exception ex) {
    if (isTimeout(ex)) {
        return ProviderResult.retriableFailure("Timeout");
    }
    return ProviderResult.nonRetriableFailure("Unexpected");
}
```
