3 大视觉原则：
1. 简约：视觉元素克制，必要层次差异，不滥用色彩；优先字体粗细 / 间距区分而非颜色堆砌
2. 易读：长文档阅读舒适，合适行高 / 段距 / 字号 / 对比度；代码块清晰，表格扫读快
3. 易改：CSS 变量集中在 :root，调一处就改全局；命名语义化（不出现 --c-1 这种位置依赖名）

5 项结构性约束（不可违反）：
A. 支持三栏布局：grid-template-columns: 220px minmax(0, 1fr) 320px; gap: 24px;
   左栏 .toc 和右栏 .notes-panel 都是 sticky (top: 24px)
   响应式 @media (max-width: 1024px) 折叠为单栏（TOC 移到顶部 details，右栏移到底部）
B. 4 类 note pill 必须视觉可区分：data-type="decision" / "change" / "tradeoff" / "other"
C. task card 的 data-status 状态机必须有显著视觉区分：pending / in-progress / done / blocked / skipped
D. HTML 单文件自包含——CSS inline，禁止外部依赖（字体可引 Google Fonts）
E. CSS 变量名必须跨两份模板保持完全一致（手工同步）

必须覆盖的组件清单：
- header.doc-header（含 meta-bar、status-pill）
- nav.toc（左栏导航）
- aside.callout（4 色：info / success / warning / danger / summary）
- table（含偶行斑马纹）
- pre + code（暗色代码块）
- .pill（语义 pill：low/mid/high/create/modify/delete/verify；类型 pill：decision/change/tradeoff/other）
- .decision（决策卡片，含 .alternatives）
- article.task + ol.steps（含 data-status 5 状态渲染）
- aside.notes-panel（右栏面板，含 ol.notes-log、empty-state、notes-stats footer）
- dl.meta（plan 头部元数据）
- aside.for-ai-worker（plan 头部 AI 工作者提示）

字体：Google Fonts (Inter + JetBrains Mono)

产出形式：仅给我完整的 <style>...</style> 内容（不写 HTML 结构、不写文件）。
我会把这段 CSS inline 到两份模板的 <head> 中。
