# Release Checklist

## Before Release
- Changelog updated with highlights and upgrade notes.
- ADRs reflect final decisions; deprecations noted.
- Contracts versioned (API/Event) and validated for compatibility.
- Test plans executed; QA sign-off complete.

## During Release
- Tag and artifact versions consistent across modules.
- Verify health checks and readiness.

## After Release
- Monitor key metrics and error budgets.
- Document incidents and follow up ADRs if needed.
