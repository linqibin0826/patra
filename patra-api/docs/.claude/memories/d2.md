# D2 声明式图表规范

> **适用场景**：复杂架构图、ERD 数据库建模、UML 类图、序列图、嵌套容器拓扑。
> **与 Mermaid 的区别**：D2 支持更强大的容器嵌套、SQL 表建模、样式类系统和多布局引擎。

---

## 强制规则（TL;DR）

1. **ID 与 Label 分离** — ID 仅用英文+下划线，Label 通过属性设置中文
2. **禁止隐式声明** — 所有节点必须显式定义 `shape`，禁止仅通过连接自动创建
3. **禁止内联样式** — 使用 `classes` 集中管理样式，禁止 `style.fill: "#xxx"` 硬编码
4. **块语法优先** — 复杂配置必须使用 `{}` 块语法，禁止单行压缩
5. **统一连接方向** — 全部使用 `->` 正向箭头，禁止 `<-` 反向箭头

## 标识符规范

### ID 与 Label 分离

**错误**：将中文或空格直接用作 ID

```d2
# ❌ 错误：ID 包含中文，修改 Label 需重构所有连接
用户终端 -> 认证网关: 发送请求
```

**正确**：ID 作为逻辑索引，Label 负责展示

```d2
# ✅ 正确：ID 稳定，Label 可随时修改
client_app: 用户终端 {
  label: 用户登录界面\n(Web/Mobile)
  shape: rectangle
}

auth_gateway: 认证网关 {
  shape: hexagon
}

client_app -> auth_gateway: 发送请求
```

### 命名规则

| 元素 | 规范 | 示例 |
|------|------|------|
| 节点 ID | `snake_case`，仅英文+数字+下划线 | `user_service`、`db_primary` |
| 容器 ID | 同上，语义化命名 | `aws_region`、`k8s_cluster` |
| 连接标签 | 动词短语 | `Authenticates`、`Reads from` |

### 保留关键字与特殊字符

当 ID 与关键字冲突或包含特殊字符时，必须用引号包裹：

```d2
# 关键字冲突
"shape": 几何图形说明 {
  shape: document  # 内部的 shape 才是属性
}

# 包含点号的文件名
"server.conf": {
  shape: file
}

"server.conf" -> app_service: 读取配置
```

**保留关键字**：`shape`、`style`、`label`、`icon`、`near`、`vars`、`classes`、`tooltip`

## 语法风格

### 块语法 vs 行内语法

| 场景 | 推荐语法 | 示例 |
|------|----------|------|
| 简单节点（仅 Label） | 行内 | `api: API 服务` |
| 包含 shape | 行内 | `db: 数据库 {shape: cylinder}` |
| 包含样式或嵌套 | **块语法** | 见下方 |

**复杂配置必须使用块语法**：

```d2
# ✅ 正确：属性垂直排列，Git Diff 清晰
database: 主数据库 {
  shape: cylinder
  label: Core DB

  style: {
    fill: "#ede9fe"
    stroke: "#8b5cf6"
    font-color: "#1e293b"
  }
}
```

## 容器与嵌套

### 显式嵌套（块语法）

**禁止**使用点标记法隐式创建深层嵌套：

```d2
# ❌ 错误：层级丢失，重构困难
aws.region_us.vpc.subnet.web_server.shape: square
```

**必须**使用块语法构建层级：

```d2
# ✅ 正确：代码缩进与图表结构同构
aws: AWS Cloud {
  shape: cloud

  region_us: US East {
    vpc: Primary VPC {
      style.stroke: "#3b82f6"

      subnet_public: Public Subnet {
        web_server: {shape: square}
      }

      subnet_private: Private Subnet {
        db_server: {shape: cylinder}
      }
    }
  }
}

# 跨层级连接使用完整路径
aws.region_us.vpc.subnet_public.web_server -> aws.region_us.vpc.subnet_private.db_server
```

### 容器形状

| 形状 | `shape` 值 | 适用场景 |
|------|-----------|----------|
| 云 | `cloud` | 云环境、外部系统 |
| 包 | `package` | 逻辑模块、命名空间 |
| 圆柱 | `cylinder` | 数据库、存储 |
| 六边形 | `hexagon` | 服务、处理器 |
| 矩形 | `rectangle` | 默认容器 |

## 连接规范

### 连接符语义

| 符号 | 含义 | 使用场景 |
|------|------|----------|
| `->` | 单向（**推荐**） | 数据流、调用关系 |
| `<->` | 双向 | 双向通信、同步 |
| `--` | 无向 | 物理连接、对等关系 |
| `<-` | 反向（**禁止**） | — |

### 连接定义规范

**禁止**链式连接与反向箭头混用：

```d2
# ❌ 错误：逻辑混乱，难以扩展
User -> LoadBalancer -- WAF <- Firewall
```

**必须**拆分为单行，统一方向：

```d2
# ✅ 正确：每行一个交互，易于添加标签
User -> LoadBalancer: HTTPS 请求
LoadBalancer -> WAF: 转发流量
WAF -> AppServer: 过滤后请求
Firewall -> WAF: 同步策略 {style.stroke-dash: 3}
```

### 连接标签

- 使用动词短语：`Reads`、`Authenticates`、`Deploys to`
- 长文本使用 `tooltip`：

```d2
ServiceA -> ServiceB: 同步用户画像 {
  tooltip: |md
    ### Payload
    - User Meta: ID, Name
    - Permissions: RBAC Roles
  |
}
```

## 样式系统

### Classes（样式类）

**禁止**内联样式硬编码：

```d2
# ❌ 错误：重复代码，修改困难
UserDB: {style: {fill: "#f4a261"; stroke: "#e76f51"}}
OrderDB: {style: {fill: "#f4a261"; stroke: "#e76f51"}}
```

**必须**使用语义化类：

```d2
# ✅ 正确：定义一次，到处使用（浅色模式友好）
classes: {
  datastore: {
    shape: cylinder
    style: {
      fill: "#ede9fe"
      stroke: "#8b5cf6"
      stroke-width: 2
      font-color: "#1e293b"
    }
  }

  service: {
    style: {
      fill: "#dcfce7"
      stroke: "#22c55e"
      stroke-width: 2
      font-color: "#1e293b"
    }
  }

  layer: {
    style: {
      fill: "#f8fafc"
      stroke: "#64748b"
      stroke-width: 2
      font-color: "#1e293b"
    }
  }
}

UserDB.class: datastore
OrderDB.class: datastore
AuthService.class: service
```

### 语义化颜色映射

| 用途 | 背景色 | 边框色 | 文字色 |
|------|--------|--------|--------|
| **容器/层** | `#f8fafc` | `#64748b` | `#1e293b` |
| **默认节点** | `#dbeafe` | `#3b82f6` | `#1e293b` |
| **服务/处理** | `#dcfce7` | `#22c55e` | `#1e293b` |
| **存储/数据** | `#ede9fe` | `#8b5cf6` | `#1e293b` |
| **可视化** | `#ffedd5` | `#f97316` | `#1e293b` |
| **告警/错误** | `#fecaca` | `#ef4444` | `#1e293b` |

### Globs（通配符）

使用过滤器精准匹配：

```d2
# 仅对矩形节点设置圆角
*: {
  &shape: rectangle
  style.border-radius: 8
}

# 匹配后缀为 _test 的节点
*_test: {
  style: {
    opacity: 0.6
    fill: lightgray
    stroke-dash: 3
  }
}
```

## 特定领域建模

### SQL 表（ERD）

**必须**使用 `shape: sql_table`：

```d2
users: {
  shape: sql_table

  id: uuid {constraint: primary_key}
  username: varchar(50) {constraint: unique}
  role_id: int {constraint: foreign_key}
  created_at: timestamp
}

roles: {
  shape: sql_table
  id: int {constraint: primary_key}
  name: varchar
}

# 字段级外键关系
users.role_id -> roles.id
```

### UML 类图

使用可见性前缀：

```d2
AuthManager: {
  shape: class

  # 属性
  -sessionPool: Map<String, Session>
  +timeout: int

  # 方法
  +validate(token: string): bool
  -cleanUp(): void
  #notifyObservers(): void
}
```

| 前缀 | 可见性 |
|------|--------|
| `+` | public |
| `-` | private |
| `#` | protected |

### 序列图

**必须**预定义参与者顺序：

```d2
shape: sequence_diagram

# 1. 强制排序（从左到右）
User: {shape: person}
UI
Server
DB: {shape: cylinder}

# 2. 交互流（从上到下）
User -> UI: 点击查询
UI -> Server: GET /api/data
Server -> DB: SQL SELECT
DB -> Server: ResultSet
Server -> UI: JSON Response
UI -> User: Render Table
```

## 布局引擎

### 引擎选择

| 引擎 | 特点 | 适用场景 |
|------|------|----------|
| `dagre` | 快速，层级分明 | 简单流程图 |
| `elk` | 专业级，支持复杂嵌套 | **微服务架构（推荐）** |
| `tala` | 商业级美学 | ERD、对称布局 |

### 指定引擎

```d2
vars: {
  d2-config: {
    layout-engine: elk
  }
}
```

## 变量与模块化

### 变量定义

```d2
vars: {
  # 使用 Tailwind CSS 调色板，浅色模式友好
  colors: {
    layer_bg: "#f8fafc"
    layer_border: "#64748b"
    service_bg: "#dcfce7"
    service_border: "#22c55e"
    storage_bg: "#ede9fe"
    storage_border: "#8b5cf6"
    text: "#1e293b"
  }
  app_version: "v2.1.0"
}

server: Web Server (${app_version}) {
  style: {
    fill: ${colors.service_bg}
    stroke: ${colors.service_border}
    font-color: ${colors.text}
  }
}
```

### 模块导入

```d2
# 导入通用样式库
...path/to/lib/styles.d2

# 导入组件作为子图
network_layer: {
  ...path/to/components/network.d2
}
```

## 代码审查清单

| 检查项 | 要求 |
|--------|------|
| 标识符 | 全英文 ID？避免中文？ |
| 显式定义 | 所有节点有 `shape`？ |
| 样式分离 | 使用 `classes`？无硬编码？ |
| 连接方向 | 全部 `->`？无 `<-`？ |
| 模块化 | 超 200 行已拆分？ |
| 语义准确 | 数据库用 `sql_table`？ |
