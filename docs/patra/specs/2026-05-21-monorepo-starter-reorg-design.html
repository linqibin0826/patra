Monorepo Starter 重组：Patra 顶层三段式布局

📅 2026-05-21 🌿 worktree-brainstorm-starter-reorg 🎯 spec draft
关键决策摘要

把堆在 patra-api/ 里的 13 个通用 starter（linqibin-*）与 2 个项目专属 starter（patra-spring-boot-starter-*）拆出，按"通用 / 项目 / 业务"三段提升到 Patra/ 顶层；Gradle 改造为单 root，使用拍平 path + projectDir 重映射（现有跨模块依赖零改动）；Maven group 收紧为两段——dev.linqibin.commons 与 dev.linqibin.patra；linqibin-commons 禁止反向依赖 patra-* 的边界靠 Gradle convention plugin 强制校验，不靠约定；单 PR 单 worktree 完成，4 个逻辑 commit 各自可构建。

1. 背景与现状

Patra 工作区由多 repo 合并成 monorepo 后，Java 后端代码堆积在 patra-api/ 一个子目录下，同时承担"业务子树"和"工作区根容器"两个职责：13 个通用 Spring Boot starter（linqibin-*，命名暗示个人通用基建）、2 个项目专属 starter（patra-spring-boot-starter-{expr,provenance}）、7 个业务模块（patra-catalog、patra-ingest 等）、build-logic/、.github/、Gradle wrapper、配置文件等全部混在一起。与之相对，patra-portal/ 和 patra-infra/ 已在 Patra/ 顶层并列，结构层级不对称。

本次重组对齐 monorepo 工作区根标准布局（参考 Gradle 官方 "Structuring Software Projects"、spring-projects/spring-boot 仓库、Bazel/Nx/Turborepo 实践）：顶层是 workspace root，配置/CI/工具/共享 build-logic 在根，业务/通用 starter/项目 starter 各为独立子树。Codex 独立评审已收纳，主要修正：group 三段收紧为两段、依赖契约改为 Gradle convention plugin 强制校验、commit 1 切分边界扩大避免中间态构建失败、验证清单系统化。

2. 范围边界

In Scope
  • linqibin-* 13 个 starter + 2 个 commons 上移到 linqibin-commons/
  • patra-spring-boot-starter-* 2 个 starter 上移到 patra-starters/
  • Gradle root（settings/build/properties/wrapper/gradle/libs.versions.toml）、build-logic/、spotbugs-exclude.xml 上移到 Patra/ 顶层
  • .github/ 上移；workflow working-directory 全扫去掉 patra-api/ 前缀
  • .editorconfig、.pre-commit-config.yaml、.sdkmanrc 上移；patra-api/.gitignore 合并入 Patra/.gitignore
  • scripts/{git,javadoc,letpub}/ 整体上移
  • 删除 patra-api/plugins/patra-codex/（弃用）
  • 新建 Patra/README.md 作 workspace 入口
  • 修正 patra-api/.run/PatraXxxApplication.run.xml 内 $PROJECT_DIR$/../patra-infra/ 为 $PROJECT_DIR$/patra-infra/
  • 新增 Gradle convention plugin 强制 linqibin-commons/* 不依赖 patra-* 的边界校验，挂在 check 任务
Out of Scope
  • patra-api/.github/workflows/ci.yml 全面 Gradle 化重写——遗留大量 hashFiles('**/pom.xml') 等 Maven 时代逻辑，留作单独后续 PR，本次仅做路径前缀修正
  • linqibin-* 抽离为独立 git repo / 发布 Maven Central——保留通路（坐标分段），但不实施
  • 业务模块（patra-catalog 等）重新分组（如 services/libs）——保持现状，仅整体上移到 patra-api/ 下并通过 projectDir 重映射
  • Gradle path 改为嵌套（:linqibin-commons:linqibin-commons-core）——使用拍平 + 重映射避免全库 sed
  • publishToMavenLocal 之外的发布配置（source jar、javadoc jar、signing、Maven Central pom 元数据）——YAGNI

归属分析框架：

  • CCP (Common Closure Principle)：一起变化的放一起。Gradle 升级 = 跨子树事件 ⇒ 配置统一根；业务 run config = 仅 patra-api 内 ⇒ 留 patra-api/.run/。
  • Group by Consumer：制品放消费者最近公共祖先。build-logic 被三方共享 ⇒ Patra/ 顶层；.run/PatraXxxApplication 仅服务 patra-api 业务 ⇒ 留 patra-api/。
  • Monorepo workspace root 通识：顶层放配置/CI/工具，子项目独立子目录（Gradle/spring-boot/Nx/Bazel 同构）。
  • SST (Single Source of Truth)：.editorconfig / .pre-commit-config.yaml / .sdkmanrc 只能仓库根有一份，否则失效或漂移。

4. Gradle 改造

4.1 单 root + 拍平 path + projectDir 重映射

核心理由：现有跨模块依赖大量使用拍平形式 api(project(":linqibin-commons-core"))，重映射方案让这些引用零改动；Gradle 官方支持修改 ProjectDescriptor.projectDir，非 hack。

// Patra/settings.gradle.kts
rootProject.name = "patra"

pluginManagement {
    includeBuild("build-logic")
    repositories { gradlePluginPortal(); mavenCentral() }
}

dependencyResolutionManagement {
    repositories {
        maven { url = uri("https://maven.aliyun.com/repository/public") }
        mavenCentral()
    }
    repositoriesMode = RepositoriesMode.FAIL_ON_PROJECT_REPOS
}

// 集中映射 helper
fun includeAt(path: String, dir: String) {
    include(path)
    project(path).projectDir = file(dir)
}

// linqibin-commons/ —— 13 starter + 2 commons
includeAt(":linqibin-commons-core", "linqibin-commons/linqibin-commons-core")
includeAt(":linqibin-commons-storage", "linqibin-commons/linqibin-commons-storage")
includeAt(":linqibin-spring-boot-starter-core", "linqibin-commons/linqibin-spring-boot-starter-core")
// ... 其余 10 个 starter 同样

// patra-starters/ —— 2 项目 starter
includeAt(":patra-spring-boot-starter-expr", "patra-starters/patra-spring-boot-starter-expr")
includeAt(":patra-spring-boot-starter-provenance", "patra-starters/patra-spring-boot-starter-provenance")

// patra-api/ —— 业务模块（嵌套 path 现状保留）
includeAt(":patra-common", "patra-api/patra-common")
includeAt(":patra-common:patra-common-model", "patra-api/patra-common/patra-common-model")
// ... 其余业务模块同样

4.2 Group 两段：显式设置，禁用 allprojects 隐式判断

linqibin-commons/* 独立身份 dev.linqibin.commons，patra-starters/* 与 patra-api/* 共享 dev.linqibin.patra（patra-starters 本质是 Patra 产品的一部分，与业务同 group 更准）。group 在各模块通过 apply 对应 convention plugin 显式设置，不使用 allprojects { group = when { projectDir.startsWith(...) } } 的隐式判断（避免目录移动时 group 静默变化、避免 configuration cache 兼容性问题）。

// gradle.properties
patraVersion=0.1.0-SNAPSHOT
linqibinCommonsGroup=dev.linqibin.commons
patraGroup=dev.linqibin.patra

// build-logic/src/main/kotlin/linqibin.module-commons.gradle.kts
plugins { id("linqibin.spring-boot-starter") }  // 或 linqibin.java-library
group = providers.gradleProperty("linqibinCommonsGroup").get()

// build-logic/src/main/kotlin/linqibin.module-patra.gradle.kts
group = providers.gradleProperty("patraGroup").get()

// 子模块 build.gradle.kts 显式 apply 身份
// linqibin-commons/linqibin-spring-boot-starter-jpa/build.gradle.kts
plugins { id("linqibin.module-commons") }
// patra-starters/patra-spring-boot-starter-expr/build.gradle.kts
plugins { id("linqibin.module-patra"); id("linqibin.spring-boot-starter") }
// patra-api/patra-catalog/patra-catalog-app/build.gradle.kts
plugins { id("linqibin.module-patra"); id("linqibin.hexagonal-app") }

4.3 依赖契约：Gradle convention plugin 强制校验

"linqibin-commons 禁止依赖 patra-*"不能靠约定，必须 fail-fast。新增 linqibin.boundary-check convention plugin，linqibin-commons/* 模块通过 linqibin.module-commons 间接自动 apply，注册 checkBoundary task 并挂入 check 生命周期：

// build-logic/src/main/kotlin/linqibin.boundary-check.gradle.kts
val checkBoundary = tasks.register("checkBoundary") {
    group = "verification"
    doLast {
        val forbidden = configurations
            .flatMap { it.dependencies }
            .filterIsInstance<ProjectDependency>()
            .map { it.dependencyProject.path }
            .filter { it.startsWith(":patra-") }
        if (forbidden.isNotEmpty()) {
            throw GradleException(
                "${project.path} (linqibin-commons) 禁止依赖 patra-* 模块: $forbidden"
            )
        }
    }
}
tasks.named("check") { dependsOn(checkBoundary) }

6. 迁移路径

单 worktree / 单 PR，逻辑切分为 4 个 commit，每个 commit 末态满足 ./gradlew clean build 全绿。

Commit 1：Gradle root 上移 + 配套基础设施（最大单步）

这是最危险断点——settings.gradle.kts 上移后，Gradle 从根目录解析 includeBuild("build-logic") 和所有 projectDir；任何路径错配都会让构建直接挂。必须在同一 commit 内同步完成以下全部内容：

  • 移动：patra-api/{settings.gradle.kts, build.gradle.kts, gradle.properties, gradlew, gradlew.bat, gradle/, spotbugs-exclude.xml} → Patra/ 顶层
  • 移动：patra-api/build-logic/ → Patra/build-logic/（其内部 settings.gradle.kts 引用 ../gradle/libs.versions.toml 路径继续成立）
  • 移动：patra-api/{.editorconfig, .pre-commit-config.yaml, .sdkmanrc} → Patra/
  • 合并：patra-api/.gitignore → Patra/.gitignore
  • 移动：patra-api/.github/ → Patra/.github/；workflow 内所有 working-directory: patra-api 去掉前缀（ci.yml 的 Maven pom.xml 残留不动，留作后续 PR）
  • 移动：patra-api/scripts/ → Patra/scripts/
  • 顶层 settings.gradle.kts 暂以 patra-api/* 路径 include 现有模块（中间态可构建）：业务模块物理位置不变、starter 物理位置不变、只是 settings 文件位置变
  • 删除：patra-api/plugins/patra-codex/

验证：./gradlew clean build 全绿；./gradlew projects 输出与 commit 前完全一致。

Commit 2：linqibin-commons/ 上移 + Group 拆分

  • git mv patra-api/linqibin-* linqibin-commons/（13 starter + 2 commons）
  • settings.gradle.kts 改用 includeAt helper + projectDir 重映射
  • build-logic 新增 linqibin.module-commons、linqibin.module-patra、linqibin.boundary-check 三个 convention plugin
  • 各子模块 build.gradle.kts 显式 apply 身份 plugin（patra-api/ 业务模块也要 apply linqibin.module-patra，否则 group 缺省）
  • 移除根 build.gradle.kts 内 allprojects { group = ... } 行

验证：./gradlew clean build 全绿；./gradlew :linqibin-spring-boot-starter-jpa:dependencies 显示 group dev.linqibin.commons；./gradlew :linqibin-spring-boot-starter-jpa:check 触发 checkBoundary 任务并通过。

Commit 3：patra-starters/ 上移

  • git mv patra-api/patra-spring-boot-starter-{expr,provenance} patra-starters/
  • settings.gradle.kts 加 2 个 includeAt 条目

验证：./gradlew clean build；./gradlew :patra-spring-boot-starter-expr:dependencies 显示 group dev.linqibin.patra。

Commit 4：清扫 + 文档

  • 新建 Patra/README.md（workspace 入口，3 子树职责 map）
  • 修正 patra-api/.run/PatraXxxApplication.run.xml（× 5 个 application，含 JRebel 副本 × 5 = 10 个文件）：$PROJECT_DIR$/../patra-infra/docker/otel-agent/... → $PROJECT_DIR$/patra-infra/docker/otel-agent/...
  • 清扫 patra-api/ 内任何残留（应只剩业务模块 + .run/ + .claude/ + CLAUDE.md + README.md）
  • publishToMavenLocal 后验证 ~/.m2/repository/dev/linqibin/{commons,patra}/ 结构

7. 关键决策与备选

决策 1：顶层切分为三段（linqibin-commons / patra-starters / patra-api）

三个语义清晰的子树，"通用基建 / 项目专属 starter / 业务"职责分离。

备选 A：二段 patra-foundation/ 容纳所有 starter（不区分通用与项目专属）—— 否决：通用 starter 未来抽离独立 repo 的通路被遮蔽；语义稍模糊。
备选 B：全扁平，每个模块都成为 Patra/ 顶层目录 —— 否决：顶层目录数膨胀到 20+，浏览压力大。

决策 2：单 Gradle root

Patra/ 顶层唯一 settings.gradle.kts，include 全部模块。IDE 一次导入即可见全部、跨子树重构与跳转无障碍。

备选 A：composite build，每个子树自有 settings + 顶层 umbrella includeBuild —— 否决：当前只有一个消费者（patra-api），独立构建无收益；IDE 跨构建重构体验劣于单 root。未来真要分离时升级到 composite 也容易。
备选 B：3 个独立 root + publishToMavenLocal —— 否决：日常迭代要手动 publish + 刷版本号，体验最差。

决策 3：拍平 Gradle path + projectDir 重映射

Gradle path 保持一级（:linqibin-commons-core），物理路径通过 project(":...").projectDir = file("linqibin-commons/...") 重映射。所有现有 api(project(":linqibin-commons-core")) 引用零改动。

备选 A：嵌套 path（:linqibin-commons:linqibin-commons-core，与物理路径一致）—— 否决：所有 build.gradle.kts 中跨模块引用需机械 sed（13 starter × N 引用点），改动面广；类型安全 accessor 全变；空中间聚合项目语义不清。spring-projects/spring-boot 走嵌套是历史选择，绿地项目可选更优解。
备选 B：混合（业务嵌套、starter 拍平）—— 否决：风格不一致，新人困惑。

决策 4：Group 两段（dev.linqibin.commons + dev.linqibin.patra）

linqibin-commons 独立身份，patra-starters 与 patra-api 共享 dev.linqibin.patra group。各模块通过 convention plugin 显式设 group，禁用 allprojects + when 隐式判断。

备选 A：三段（linqibin-commons / dev.linqibin.patra.starters / dev.linqibin.patra）—— 否决：Codex 评审认为过度设计，starters 与 patra 业务本质同 group；projectDir.startsWith 隐式分派脆弱。
备选 B：全仓一段（dev.linqibin.patra）—— 否决：失去 linqibin-commons 独立身份信号；未来抽离需一次性改 group，工作量与现在分两段近似。

决策 5：依赖边界靠 Gradle convention plugin 强制，不靠约定

新增 linqibin.boundary-check convention plugin，linqibin-commons/* 模块自动注册 checkBoundary task，遍历所有 ProjectDependency，若依赖 :patra-* 则 fail 构建。挂入 check 生命周期。

备选 A：靠 README / CONTRIBUTING 文档约定 —— 否决：单人项目也会自己手滑；fail-fast 是绿地项目"质量优先"原则的具体落地。
备选 B：依赖 group 隔离做约束 —— 否决：group 不阻止 project(":...") 引用，错位语义。

决策 6：单 PR / 单 worktree，4 个逻辑 commit

原子性强，中间态不发布；commit 内部按"基础设施上移 → linqibin-commons 上移 → patra-starters 上移 → 清扫"逻辑切分，每个 commit 末态可构建。

备选 A：3 个独立 PR 分批 —— 否决：单人项目无并行需求；中间态需保证可构建，对 settings 反复折腾。
备选 B：单 commit 一刀切 —— 否决：grep / bisect 困难，commit 1 的高风险变更与 commit 2 的纯文件移动应该可区分。

决策 7：CI 重写延后到单独 PR

本次仅做 .github/ 上移与 workflow 内 working-directory 路径前缀修正。ci.yml 中 hashFiles('**/pom.xml') 等 Maven 时代遗留全面重写为 Gradle 版本作为下一个 PR 范围。

备选 A：本次顺手重写 ci.yml —— 否决：扩大 PR 商务范围，review 复杂度上升；CI 重写本身是独立工作单元（牵涉 spotless / jacoco / boundary task 等任务编排），值得单独 spec。
备选 B：不动不送，作 known issue —— 否决：CI 上移后路径前缀失配会立即红，至少要修这一项。

8. 风险与缓解

风险等级缓解
Commit 1 中间态不可构建：Gradle root 上移后 build-logic 找不到、libs.versions.toml 路径错位、workflow working-directory 残留 patra-api 前缀 high Commit 1 严格打包：settings/build/properties/wrapper/gradle 完整一次性移动；build-logic/settings.gradle.kts 内 ../gradle/libs.versions.toml 相对路径保留可用；workflow grep 扫除 working-directory: patra-api 残留；commit 末态执行 ./gradlew clean build + ./gradlew projects 双重验证
IntelliJ .run/ 配置中 $PROJECT_DIR$/../patra-infra/... 失效——根目录上移后 $PROJECT_DIR$ 变成 Patra/，原有相对路径多了一层 ../ mid Commit 4 显式修正 5 个 PatraXxxApplication + 5 个 JRebel 副本共 10 个 .run XML，将 $PROJECT_DIR$/../patra-infra/ 改为 $PROJECT_DIR$/patra-infra/；同时验证 <module name="patra.patra-xxx-boot.main"> 字段不变（rootProject.name = "patra" 不变 + 业务嵌套 path 不变 ⇒ Gradle module 名不变）
