PAP-15 — 分支保护 + Dependabot 实现计划

目标
给 Patra 仓库的 main 分支加分支保护、绑定 Required Status Check、配置 Dependabot 自动开 PR；本地一次性运行保护规则脚本完成部署。
架构
三个仓库内产物（dependabot.yml / setup-branch-protection.sh / ci.yml 加哨兵 job），PR merge 后本地手工跑一次脚本设置分支保护，最后按 AC D5 跑四步验证。
技术栈
GitHub Actions YAML / GitHub Branch Protection REST API / gh CLI / bash
关联 spec
PAP-15 — main 分支保护 + Required Status Checks + Dependabot 分组 PR

    runs-on: ubuntu-latest

任务 3：创建 scripts/setup-branch-protection.sh

创建

  1. 步骤 1：确认 scripts/ 目录存在 
    ls -la scripts/ 2>&1 | head -5

    预期：目录存在（仓库根已有 scripts/，含其他脚本如 setup-jdk-mise.sh 等）。若不存在则先 mkdir -p scripts。

  2. 步骤 2：写完整脚本到 scripts/setup-branch-protection.sh

    完整内容（来自 spec §4）：

    #!/usr/bin/env bash
# main 分支保护规则一次性配置脚本（幂等，可重复运行）
# 用法: ./scripts/setup-branch-protection.sh
# 依赖: gh CLI 已登录、对仓库有 admin 权限

set -euo pipefail

OWNER_REPO="linqibin0826/patra"
BRANCH="main"

echo "→ 配置 ${OWNER_REPO} 分支 ${BRANCH} 的保护规则..."

gh api \
  --method PUT \
  -H "Accept: application/vnd.github+json" \
  "/repos/${OWNER_REPO}/branches/${BRANCH}/protection" \
  --input - <<'JSON'
{
  "required_status_checks": {
    "strict": true,
    "contexts": ["required-check"]
  },
  "enforce_admins": false,
  "required_pull_request_reviews": null,
  "restrictions": null,
  "required_linear_history": true,
  "allow_force_pushes": false,
  "allow_deletions": false,
  "block_creations": false,
  "required_conversation_resolution": true,
  "lock_branch": false,
  "allow_fork_syncing": false
}
JSON

echo "✓ 完成。当前规则:"
gh api "/repos/${OWNER_REPO}/branches/${BRANCH}/protection" \
  | jq '{required_status_checks, enforce_admins, required_linear_history, allow_force_pushes, allow_deletions, required_conversation_resolution}'
  3. 步骤 3：chmod +x 并校验 
    chmod +x scripts/setup-branch-protection.sh
ls -la scripts/setup-branch-protection.sh

    预期：权限位含 x（如 -rwxr-xr-x）。

  4. 步骤 4：bash 语法校验 + shellcheck（如果安装了） 
    bash -n scripts/setup-branch-protection.sh && echo "bash syntax OK"
shellcheck scripts/setup-branch-protection.sh 2>&1 || echo "shellcheck 未安装或有 lint 警告，可接受"

    预期 bash -n 输出 bash syntax OK。shellcheck 是可选 lint，可能对 set -euo pipefail 之类有建议，对功能不影响。

    不要在本步骤实际运行脚本——脚本会真的 PUT 分支保护规则；运行时机在 Task 5（PR merge 之后）。

  5. 步骤 5：Commit

git commit -m "$(cat <<'EOF'
