PAP-10 实施计划：CI 全面 Gradle 化 + JVM Test Suites + JaCoCo 聚合 + Codecov

📅 2026-05-21 🌿 worktree-pap-10-ci-gradle-jacoco 📋 plan draft
面向 AI 代理的工作者： 必需子技能：使用 patra:subagent-driven-development（推荐） 或 patra:executing-plans 逐任务实现。 通过修改任务卡片与步骤的 data-status 属性追踪进度 （pending / in-progress / done / blocked / skipped）。 每完成一个 task 立即 commit。
目标
把 .github/workflows/ 从已失效的 Maven 命令链改造为 Gradle，同时把测试结构升级到 JVM Test Suites（unit / integrationTest / e2eTest 三层物理隔离），并通过 jacoco-report-aggregation 聚合覆盖率上传 Codecov。
架构
在 linqibin.java-base convention plugin 中统一声明 JVM Test Suites，所有 Java 模块自动获得三个 source set；54 个 *IT.java + 3 个 *E2E.java + 7 个 *ContainerInitializer.java 用 git mv 迁移到对应目录。e2eTest source set 通过 compileClasspath 继承 integrationTest 的 output（共享 Initializer）；integrationTest 配置 extendsFrom 继承 testImplementation 全部依赖。CI 用单 workflow ci.yml + 单 job build-and-test，靠 gradle/actions/setup-gradle@v4 build cache 加速。
技术栈
Gradle 9.5 / JVM Test Suites (incubating since Gradle 7.3) / jacoco-report-aggregation (Gradle 7.4+) / GitHub Actions / gradle/actions/setup-gradle@v4 / codecov/codecov-action@v4 / Testcontainers (postgres:17, apache/rocketmq:5.3.1) / JUnit 5

// ==================== JVM Test Suites ====================
testing {
    suites {
        val test by getting(JvmTestSuite::class) {
            useJUnitJupiter()
            targets.configureEach {
                testTask.configure {
                    useJUnitPlatform {
                        excludeTags("manual")
                        if (!project.hasProperty("runExternal")) excludeTags("external")
                    }
                    maxParallelForks = (Runtime.getRuntime().availableProcessors() / 2).coerceAtLeast(1)
                    jvmArgs("--add-opens", "java.base/java.lang=ALL-UNNAMED")
                    testLogging { events("passed", "skipped", "failed") }
                }
            }
        }

        register<JvmTestSuite>("integrationTest") {
            useJUnitJupiter()
            dependencies {
                implementation(project())
            }
            targets.configureEach {
                testTask.configure {
                    shouldRunAfter(test)
                    useJUnitPlatform {
                        excludeTags("manual")
                        if (!project.hasProperty("runExternal")) excludeTags("external")
                    }
                    maxParallelForks = (Runtime.getRuntime().availableProcessors() / 2).coerceAtLeast(1)

            }
        }

        register<JvmTestSuite>("e2eTest") {
            useJUnitJupiter()

// ==================== Test Suite 之间的配置/source 共享 ====================
// integrationTest 与 e2eTest 各自的 Implementation/RuntimeOnly/CompileOnly 配置
// 默认是独立的；这里让它们继承 test 的 declarative 依赖，避免每模块重复声明。
listOf("integrationTest", "e2eTest").forEach { suiteName ->
    configurations.named("${suiteName}Implementation") {
        extendsFrom(configurations.named("testImplementation").get())
    }
    configurations.named("${suiteName}RuntimeOnly") {
        extendsFrom(configurations.named("testRuntimeOnly").get())
    }
    configurations.named("${suiteName}CompileOnly") {
        extendsFrom(configurations.named("testCompileOnly").get())
    }
}

// e2eTest 复用 integrationTest source set 输出（共享 *ContainerInitializer 等基础类）
sourceSets {
    named("e2eTest") {
        compileClasspath += sourceSets["integrationTest"].output
        runtimeClasspath += sourceSets["integrationTest"].output
    }
}
