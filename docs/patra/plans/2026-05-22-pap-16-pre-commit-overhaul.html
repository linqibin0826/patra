任务 6：首次全仓清扫 + 单独 commit

commit
hygiene hooks 对全仓首次跑修出的改动
  1. 步骤 1：跑 pre-commit run --all-files

    运行：

    pre-commit run --all-files

    预期：

    • 大部分 hook 报 Passed
    • 少数 hook（end-of-file-fixer / trailing-whitespace / mixed-line-ending）可能报 Failed + files were modified by this hook，并列出修改的文件清单
    • spotless-apply：因为已存在的 Java/gradle.kts 文件改动量 0（hook 配置 files 过滤后无匹配），Skipped（输出 (no files to check) Skipped）。这是符合预期的——pre-commit run --all-files 会忽略 files 过滤,但 spotless local hook 的 entry 是 ./gradlew spotlessApply -q --no-daemon,Gradle 任务级缓存会使其极快(秒级)。如果 spotlessApply 修了文件,也算清扫的一部分

    耗时：首次约 1–3 分钟（gitleaks 全仓扫描最慢）。

  2. 步骤 2：查看 working tree 改动

    运行：

    git status --short | head -30
echo "---total changed files:"
git status --short | wc -l

    预期：列出被 hygiene 修过的文件清单（可能数十个）。不预期改动巨大——若超过 200 个文件，停下来确认是否有非预期的 hook（比如 mixed-line-ending 误改了二进制）。

  3. 步骤 3：检查改动性质（防回归）

    抽 3-5 个文件看 diff：

    git diff --stat | head -20

    预期：每个文件改动 1–10 行，主要是末尾换行、行尾空格、CRLF→LF 等无害改动。

    若发现 非预期改动（如 .png/.jar 等二进制被改）：

    git checkout -- <binary-file>

    并在 .pre-commit-config.yaml 给对应 hook 加 exclude: '\.(png|jar|...)$'。本 plan 不预期此情况，若出现属 Implementation Note 记录在右栏。

  4. 步骤 4：commit 清扫改动

    运行：

    git add -A

`pre-commit run --all-files` 首次跑出的 hygiene 改动：
- end-of-file-fixer：补齐文件末尾换行
- trailing-whitespace：删行尾空格
- mixed-line-ending：CRLF/CR → LF

历史代码累积，单独成 commit 避免与 PAP-16 主功能 commit 的 diff 混淆。

Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>
EOF
)"

    预期：pre-commit hooks 再跑一遍，全绿（幂等）；commit 创建。
