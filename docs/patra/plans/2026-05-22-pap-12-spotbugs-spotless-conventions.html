PAP-12 + PAP-13: SpotBugs / Spotless Convention Plugins 实现计划

目标
把 SpotBugs 与 Spotless 从 linqibin.java-base 抽出为独立 convention plugins，对齐 PAP-12 AC 参数，全量基线化代码格式与告警，并把 ./gradlew check 接入 CI 作为 PR 门禁。
架构
两个独立的 Gradle precompiled script plugin 文件（linqibin.spotbugs.gradle.kts / linqibin.spotless.gradle.kts）放在 build-logic/src/main/kotlin/；linqibin.java-base 通过 id("linqibin.spotbugs") + id("linqibin.spotless") 引入，所有 Java 模块沿现有链路继承。ci.yml 单 step ./gradlew check 同时触发 test + spotbugsMain + spotlessCheck。
技术栈
Gradle 9.2.1 / Kotlin DSL / SpotBugs Gradle plugin 6.4.8（核心 4.9.3）/ Spotless 7.0.4（googleJavaFormat 1.29.0 / ktlint 1.5.0）/ GitHub Actions actions/upload-artifact@v4

Task 1：新建 linqibin.spotbugs.gradle.kts

创建
build-logic/src/main/kotlin/linqibin.spotbugs.gradle.kts
  1. 步骤 1.1：写入完整 plugin 文件

    注意：用 tasks.matching { ... }.configureEach { enabled = false } lazy filter 禁用所有非 main 的 spotbugs task。原因：spotbugsIntegrationTest / spotbugsE2eTest 由 java-base 的 JVM Test Suites 注册 sourceSet 之后才被 spotbugs plugin 创建；在本 plugin body 执行时还不存在，tasks.named("spotbugsIntegrationTest") 会抛 UnknownDomainObjectException。matching + configureEach 对未来注册的 task 也生效。

    /**
 * SpotBugs Convention Plugin
 *
 * 应用范围：所有 Java 模块（通过 linqibin.java-base 间接 apply）。
 *
 * 配置：
 * - effort=MAX / reportLevel=LOW / ignoreFailures=false
 * - 锁 toolVersion=4.9.3（与 plugin 6.4.8 配套）
 * - 只跑 spotbugsMain，禁用 spotbugsTest / spotbugsIntegrationTest / spotbugsE2eTest 等
 * - excludeFilter 沿用根目录 spotbugs-exclude.xml
 * - HTML + XML 双报告
 *
 * 关联：PAP-12 + 决策 C（spotbugsTest 不阻断）。
 */

import com.github.spotbugs.snom.Confidence
import com.github.spotbugs.snom.Effort
import com.github.spotbugs.snom.SpotBugsTask

plugins {
    java
    id("com.github.spotbugs")
}

val libs = the<org.gradle.api.artifacts.VersionCatalogsExtension>().named("libs")

spotbugs {
    toolVersion = libs.findVersion("spotbugs").get().requiredVersion
    effort = Effort.MAX
    reportLevel = Confidence.LOW
    ignoreFailures = false
    excludeFilter = rootProject.file("spotbugs-exclude.xml")
}

// 决策 C + PAP-12 AC：不分析 test 代码。
// 用 lazy filter + configureEach，覆盖所有 non-main spotbugs task。
// 原因：spotbugsIntegrationTest / spotbugsE2eTest 由 JVM Test Suites 注册的 sourceSet
// 派生而来，在本 plugin body 执行时尚未注册，tasks.named() 会抛 UnknownDomainObjectException。
// matching + configureEach 是 lazy，对未来注册的 task 也生效。
tasks.matching { it.name.startsWith("spotbugs") && it.name != "spotbugsMain" }
    .configureEach { enabled = false }

// 仅对 spotbugsMain 配置 HTML + XML 双报告
tasks.named<SpotBugsTask>("spotbugsMain") {
    reports.create("html") { required = true }
    reports.create("xml") { required = true }
}
  2. 步骤 1.2：跑 Gradle help 验证 plugin 编译通过

    此时还未在任何模块 apply，./gradlew help 只验证 plugin 本身 Kotlin DSL 编译无错。

    运行：./gradlew help --no-daemon

    预期：退出码 0；输出含 BUILD SUCCESSFUL；不能出现 e: file:...linqibin.spotbugs.gradle.kts 编译错误。

  3. 步骤 1.3：Commit 
    git add build-logic/src/main/kotlin/linqibin.spotbugs.gradle.kts
git commit -m "build(spotbugs): 新增 linqibin.spotbugs convention plugin

抽出 SpotBugs 配置为独立 convention plugin，对齐 PAP-12 AC：
- effort=MAX / reportLevel=LOW / ignoreFailures=false
- toolVersion=4.9.3（与 plugin 6.4.8 配套）
- 显式禁用 spotbugsTest / spotbugsIntegrationTest / spotbugsE2eTest
- HTML + XML 双报告

下一步：Task 2 抽 Spotless / Task 3 java-base 引入。"

Task 2：新建 linqibin.spotless.gradle.kts

创建
build-logic/src/main/kotlin/linqibin.spotless.gradle.kts
  1. 步骤 2.1：写入完整 plugin 文件

    注意：googleJavaFormat 默认 2 空格，与 .editorconfig indent_size = 2 对齐。PAP-13 release-spec AC 写的"4 空格"为口误（Task 8 同步修正）。

    /**
 * Spotless Convention Plugin
 *
 * 应用范围：所有 Java 模块（通过 linqibin.java-base 间接 apply）。
 *
 * Java 格式化规则：
 * - googleJavaFormat 1.29.0（默认 2 空格缩进，与 .editorconfig 对齐）
 * - importOrder() 规范 import 顺序
 * - removeUnusedImports / trimTrailingWhitespace / endWithNewline
 * - 仅 src/**\/*.java，排除 build 与 generated 产物
 *
 * Gradle 脚本规则：
 * - kotlinGradle 覆盖根目录 + build-logic 下所有 .gradle.kts
 * - ktlint 1.5.0
 *
 * 关联：PAP-13 + 决策 C（spotlessCheck 阻断 PR）。
 */

plugins {
    java
    id("com.diffplug.spotless")
}

val libs = the<org.gradle.api.artifacts.VersionCatalogsExtension>().named("libs")

spotless {
    java {
        target("src/**/*.java")
        targetExclude("**/build/**", "**/generated/**")
        googleJavaFormat(libs.findVersion("google-java-format").get().requiredVersion)
        importOrder()
        removeUnusedImports()
        trimTrailingWhitespace()
        endWithNewline()
    }
    kotlinGradle {
        target("*.gradle.kts", "build-logic/**/*.gradle.kts")
        ktlint(libs.findVersion("ktlint").get().requiredVersion)
        trimTrailingWhitespace()
        endWithNewline()
    }
}
  2. 步骤 2.2：跑 Gradle help 验证编译

    运行：./gradlew help --no-daemon

    预期：退出码 0；无 linqibin.spotless.gradle.kts 相关 Kotlin DSL 编译错误。

  3. 步骤 2.3：Commit 
    git add build-logic/src/main/kotlin/linqibin.spotless.gradle.kts
git commit -m "build(spotless): 新增 linqibin.spotless convention plugin

抽出 Spotless 配置为独立 convention plugin，对齐 PAP-13 AC：
- googleJavaFormat 默认 2 空格（与 .editorconfig 一致）
- importOrder / removeUnusedImports / trimTrailingWhitespace / endWithNewline
- targetExclude **/build/** + **/generated/** 不格式化产物
- kotlinGradle 扩展到 build-logic/**/*.gradle.kts
