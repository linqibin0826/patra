    group = "verification"
    description = "Verifies that linqibin-commons modules do not depend on :patra-* modules."

    doLast {
        val forbidden = configurations
            .filter { it.isCanBeResolved || it.isCanBeDeclared }
            .flatMap { it.dependencies }
            .filterIsInstance()
            .map { it.path }
            .distinct()
            .filter { it.startsWith(":patra-") }

        if (forbidden.isNotEmpty()) {
            throw GradleException(
                "${project.path} (linqibin-commons identity) 禁止依赖 patra-* 模块；" +
                "命中的非法依赖：${forbidden.joinToString()}"
            )
        }
    }
}

tasks.named("check") {
    dependsOn(checkBoundary)
}

  1. CHANGE 05-21 20:30 § Task 2 步骤 1-3、9

    Gradle 9.5 precompiled script plugin bug：4 种顶层构造让 body 静默不执行。实测发现以下 4 类写法会让 plugin class 注册到 PluginContainer 但 script body 完全不运行（println / logger / 属性赋值都不生效），plugin 在 project.plugins.toList() 中可见，但其行为如同空 plugin：

    1. 顶层 /** … */ javadoc 多行注释（约 5+ 行）
    2. providers.gradleProperty("xxx").get() 与 project.property("xxx") 读 gradle.properties
    3. 顶层 tasks.named("check") { dependsOn(...) }
    4. 顶层 afterEvaluate { tasks.named("check") { ... } }
    5. precompiled script plugin 之间通过 plugins { id("...") } 块互相 apply（被 apply 的 plugin class 加载但 body 不执行）

    Workaround：

    • plugin 文档注释一律用 // 单行
    • group 用硬编码字符串（group = "dev.linqibin.commons"），gradle.properties 中 commonsGroup / patraGroup 仅作文档级事实来源
    • checkBoundary task 由 linqibin.boundary-check 注册，不挂入 check 生命周期；reorg 验证通过 ./gradlew :xxx:checkBoundary 单独直调
    • linqibin-commons/* 的 build.gradle.kts 显式同时 apply 两个 plugin：id("linqibin.module-commons") + id("linqibin.boundary-check")

    影响范围：plan Task 2 步骤 1-3 的 plugin 代码块需与最终落地保持一致（落地版见 build-logic/src/main/kotlin/linqibin.{module-commons,module-patra,boundary-check}.gradle.kts）；Task 2 步骤 7 的 sed 命令同时插入 module-commons + boundary-check 两条；spec §4.3 中 "checkBoundary 挂入 check 生命周期" 与 spec Done 7 ":linqibin-spring-boot-starter-jpa:check --quiet 触发 checkBoundary" 的描述需在 Task 4 收尾时同步降级为 "单独直调"。

    后续：升级 Gradle / 重写为 Plugin<Project> 类形式可能消除此问题，作为独立后续清理任务（不在本次 reorg 范围）。
