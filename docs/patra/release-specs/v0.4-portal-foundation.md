# v0.4 Portal Foundation: FE 工程基线 + Patra Design System + 门户首页

> **目标**：让 patra-portal 长出第一张"医学文献门户首页"的脸——FE 工程基线 + 用 Claude Design 建立的 Patra design system + 基于该 system 的首页 hi-fi 设计稿，并把它们全部实现到 patra-portal 中（占位 mock 数据）。
> **Linear Project**：[v0.4 Portal Foundation](https://linear.app/papertrace/project/v04-portal-foundation-fb49b56d6331)（In Progress，2026-05-23 启动）
> **Issue 池**：PAP-26 ~ PAP-34（9 个 Issue，状态 Backlog）
> **创建日期**：2026-05-22
> **上一版**：[v0.3-dev-ready](v0.3-dev-ready.md)（工程基建：CI Gradle / SpotBugs / Spotless / CodeRabbit / 分支保护 / 测试基线）

## 1. 版本目标 / 价值

访客打开 Patra 门户首页，能看到一个响应式、视觉完整、**按 hi-fi 设计稿落地**的医学文献门户首页（占位 mock 数据）；这一版同时交付 **FE 工程基线**、**用 Claude Design 建立的 Patra design system**、以及 **基于该 system 产出的首页 hi-fi 设计稿**。

**验证目标**：
- 用 Claude Design 跑通"设计系统 + 页面 hi-fi → Claude Code 实现"的完整工作流
- 给后续业务功能（搜索结果页、文献详情页、AI 解读、评论 等）接入提供视觉与组件的统一底座
- patra-portal 拥有独立的 FE CI 强制门禁

## 3. 范围边界（不做的事）

| # | 不做的事 | 理由 |
|---|---------|------|
| A | 后端真实 API / FE-BE 联调 | 数据全 hardcoded mock；BE 端点和联调留 **v0.5** |
| B | 用户系统（登录 / 注册 / 收藏提交 / 评论提交） | 卡片上"⭐ 收藏 / 💬 12"等是静态 mock 显示，不触发后端 |
| C | 搜索结果页 / 文献详情页 / 主题聚合页 / 期刊聚合页 | 只做首页；顶部 nav 其他 tab `disabled`，卡片"📖 详情"不可点 |
| D | AI 文献解读真实调用 | 卡片上"AI 速读"行是 mock 文案，不调任何 LLM；真正接入留 **v0.5+** |
| E | 搜索框真实 submit | 回车 / 点搜索仅弹"功能即将上线"toast，不跳转、不调 API |
| F | i18n / 暗色模式 / 主题切换 | v0.4 单一**中文 light theme**；i18n / dark mode 留 **v0.6+** |
| G | 独立 Storybook 文档站 / Lighthouse 性能优化 / 深度 a11y 审计 | 公共组件有 Vitest 单测即可；perf / a11y / Storybook 留后续 |

## 4. 关键决策

### A. Claude Design 产物 handoff：Prompt 直发主导 + zip 快照备份

- **影响 Issue**：PAP-26 PAP-29 PAP-30 PAP-31 PAP-32 PAP-33 PAP-34
- **判断规则**：跨 ≥2 Issue + 不可逆（一旦定下后续 design 资产都按此流程管理）
- **决策**：
  - **日常工作流**：实现 Issue 时，用 Claude Design 的 "Send to local coding agent" 模式生成 prompt + anthropic.com URL；粘到 Claude Code 后由 WebFetch 实时拉取设计文件并实现
  - **里程碑备份**：Issue #1（design system 定稿）和 Issue #2（首页 hi-fi 定稿）完成时，各下载一次 zip 快照入 `docs/patra/design/snapshots/<YYYY-MM-DD>-<topic>.zip`
  - **目的**：日常顺手（一键调用）+ 关键节点可追溯（zip 入 git，可 diff / 审计 / 断网可用）

### B. Tailwind v4 主题策略：`@theme` 块 + CSS variables

- **影响 Issue**：PAP-30 PAP-31 PAP-32 PAP-33
- **判断规则**：跨 ≥2 Issue + 不可逆（Tailwind v4 新主题机制，回到 v3 风格需要全部重写）
- **决策**：
  - 使用 Tailwind v4 的 **CSS-based `@theme` 指令**，不再用 `tailwind.config.ts`
  - Patra design tokens 写到 `patra-portal/src/styles/tokens.css`
  - 在 `src/app/globals.css` 的 `@theme` 块内桥接 tokens（如 `--color-primary: var(--patra-primary)`），供 Tailwind utility class 消费

### C. 响应式断点策略：Tailwind 默认 + 双视图（md 切分）

- **影响 Issue**：PAP-31 PAP-32 PAP-33
- **判断规则**：跨 ≥2 Issue（公共组件 + 首页 + 响应式都按此约束）
- **决策**：
  - 使用 **Tailwind 默认断点**（sm / md / lg / xl / 2xl），不引入 patra 自定义断点
  - UI 仅显式区分两类视图：**desktop ≥768px (md+)** vs **mobile <768px**
  - 移动端折叠规则统一：汉堡菜单 / 主题广场横滑 / Hero 紧凑

### D. 测试金字塔：组件 Vitest + 首页 1 条 Playwright

- **影响 Issue**：PAP-31 PAP-34
- **判断规则**：跨 ≥2 Issue + 不可逆（一旦定测试策略后续 FE Issue 都受约束）
- **决策**：
  - **公共组件 Vitest 单测**（覆盖关键 props / states，**不追求覆盖率门槛**）
  - **首页 1 条 Playwright e2e smoke test**（验证 `/` 打开 + 6 个核心区块渲染）
  - ❌ 不做：visual regression / Storybook stories / a11y 自动化审计
