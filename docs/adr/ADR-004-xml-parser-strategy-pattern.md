# ADR-004: XML 解析器策略模式重构

## 状态
已采纳

## 背景（2-3 句话）
XmlParserAdapter 原有 ~1800 行代码，包含 5 种 MeSH 记录类型的解析逻辑，难以维护和扩展。未来需支持更多 XML 类型（supp 文件、出版物 XML），硬编码方式不可持续。

## 决策（1-2 句话）
采用策略模式重构，将各记录类型的解析逻辑分离为独立策略类，XmlParserAdapter 退化为门面类。

## 原因（3-5 个要点）
1. **单一职责**：每个策略只负责一种记录类型，易于理解和测试
2. **开闭原则**：新增 XML 类型只需添加策略，无需修改门面
3. **可复用性**：EntryTermParsingStrategy 可被 DescriptorParsingStrategy 内部复用
4. **可测试性**：策略可独立单元测试，无需启动完整解析流程

## 后果（2-4 个要点）
- ✅ XmlParserAdapter 从 1800 行精简到 156 行（-92%）
- ✅ 新增 77 个策略单元测试，覆盖所有字段解析
- ⚠️ 新增 10+ 个类文件，需要理解策略模式结构
