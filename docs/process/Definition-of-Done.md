# Definition of Done (DoD)

## Every Story/PR
- ADR created/updated when architecture decisions are affected.
- Service README updated (responsibilities, contracts, runbooks links).
- API/Event contracts updated or explicitly confirmed unchanged.
- Tests implemented per Test Plan; CI green locally.
- Observability: logs/metrics/traces included for new flows.
- Changelog updated under Unreleased.

## Exit Criteria
- Acceptance criteria verified by Test Engineer.
- No open critical defects; risks documented.
- Rollback plan and operational impact noted in runbooks.
