# patra-egress-gateway 核心需求

## 需求 1：外部服务调用透传
- 业务方通过 Feign 调用网关接口，传递完整请求参数（URL、HTTP方法、Headers、Body）
- 网关将认证信息和请求参数原样透传给外部服务
- 不对响应内容进行业务数据转换或处理

## 需求 2：弹性配置管理
- 系统级配置（max + default）从 YAML 加载
- 业务方可在请求中覆盖配置，但不超过系统最大值
- 支持带配置参数和不带配置参数两种调用方式

## 需求 3：配置源演进支持
- 定义统一的 ConfigPort 接口，隔离配置来源
- 初期从 YAML 加载，后期支持数据库加载
- 切换配置源无需修改业务逻辑代码

## 需求 4：统一响应语义封装
- ResponseEnvelope 包含：
  - 成功/失败标识（基于 HTTP 状态码 2xx）
  - HTTP 状态码
  - 白名单过滤后的响应头
  - 原始响应 Body + Body 哈希值
  - 限流状态（网关自身 + 外部服务）
  - 重试建议（是否可重试、延迟时间）

## 需求 5：弹性能力实现
- **限流**：达到阈值返回 429，使用全局配置兜底
- **重试**：支持指数退避，可重试状态码（429/5xx）
- **熔断**：失败率超阈值触发熔断（503），支持半开状态
- **超时**：超时返回 504

## 需求 6：可观测性
- 日志前缀：`[EGRESS][LAYER]`
- 记录请求/响应/错误/弹性事件日志
- 敏感信息脱敏（Authorization、API-Key等）
- Trace ID 贯穿全链路

## 需求 7：错误处理与标准化
- 错误码规范：`EGR-<HTTP_STATUS>` 或 `EGR-<BUSINESS_CODE>`
- HTTP 对齐错误：0400/0422/0429/0500/0503/0504
- 业务错误：1001-1004
- 所有错误统一输出 ProblemDetail 格式（RFC 7807）

## 需求 8：多类型外部服务支持
- 通用的 HTTP 调用封装，不依赖具体服务类型
- 新增外部服务类型无需修改核心逻辑

## 需求 9：职责边界约束
- 只负责外部服务调用和弹性能力
- 不进行业务数据转换、业务规则判断、业务数据持久化
